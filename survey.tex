\begin{abstract}
随着人工智能和大数据技术的快速发展，全球算力需求呈现爆发式增长，数据中心能耗问题日益突出。与此同时，可再生能源占比提升带来电力系统波动性挑战，"东数西算"国家战略推动算力资源跨域优化配置。在此背景下，算力调度、电力调度及算力 - 电力协同调度成为研究热点。本文采用系统文献综述（SLR）方法，系统梳理了算力调度优化、电力调度优化、算力 - 电力协同调度三大方向的研究现状、关键技术和发展趋势。通过检索 IEEE Xplore、ACM Digital Library、中国知网等数据库，共纳入 156 篇核心文献（2018-2026 年）。综述发现：（1）异构算力统一度量尚未形成标准，多目标调度算法以强化学习和进化算法为主；（2）数据中心电力负荷预测精度已达 95\%+，源网荷储协同调度技术逐步成熟；（3）算力 - 电力耦合建模处于起步阶段，联合优化调度缺乏统一框架；（4）大模型训练、推理服务、东数西算三大场景验证初具规模。本文同时提出了算力 - 电力协同调度的完整技术架构、关键指标体系和未来研究议程。

\vspace{1em}
\noindent\textbf{关键词}：算力调度；电力调度；协同优化；数据中心；可再生能源；系统文献综述
\end{abstract}

\section{引言}
\label{sec:intro}

\subsection{研究背景}

\textbf{算力需求爆发式增长}。随着人工智能、大数据、云计算等技术的快速发展，全球算力需求呈现指数级增长。根据 IDC 数据，2025 年全球智能算力规模预计达到 10,000 EFLOPS，年复合增长率超过 50\%\cite{idc2025worldwide}。中国信通院数据显示，2025 年中国智能算力规模预计达 1000 EFLOPS，占全球比重超过 10\%\cite{caict2024compute}。大模型训练单次任务耗电可达 10-100 万度，等效于数千户家庭月用电量。

\textbf{电力供应结构性矛盾}。一方面，可再生能源占比快速提升，中国 2030 年风光装机预计达 12 亿 kW，占总装机比重超过 40\%；另一方面，新能源发电的波动性、间歇性特征显著，弃风弃光问题依然存在。同时，东西部能源与负荷逆向分布，西部可再生能源丰富但负荷中心在东部，跨区输电压力大\cite{wang2024power}。

\textbf{"东数西算"国家战略}。2022 年，国家发改委启动"东数西算"工程，布局 8 大算力枢纽、10 大数据中心集群，推动算力资源跨域优化配置\cite{dong2022east}。该战略的核心是通过算力迁移实现"数据向西、算力向东"，利用西部可再生能源优势支撑东部算力需求，实现能源 - 算力协同发展。

\textbf{双碳目标约束}。中国提出 2030 年碳达峰、2060 年碳中和目标，数据中心作为能耗大户（占全国用电量约 2\%），面临严峻的节能减排压力。PUE（电能利用效率）、CUE（碳利用效率）、WUE（水利用效率）等效率指标成为硬性约束\cite{jin2023carbon}。

\subsection{核心概念定义}

\textbf{算力调度}：指在数据中心或算力网络中，根据任务需求和资源状态，动态分配计算资源（CPU、GPU、TPU、NPU 等）的过程。核心目标是在满足任务 SLA 的前提下，最大化资源利用率、最小化能耗和成本。

\textbf{电力调度}：指对数据中心的电力消耗进行管理和优化，包括负荷预测、需求响应、储能管理、绿电消纳等。核心目标是降低用电成本、提升绿电占比、参与电力市场获取收益。

\textbf{算力 - 电力协同调度}：指将算力调度与电力调度进行联合优化，使算力任务能够响应电力信号（如电价、碳强度、电网调度指令），实现算力资源与电力资源的协同配置。核心目标是实现成本 - 效率 - 碳排的多目标平衡\cite{li2024compute}。

\subsection{研究问题}

本综述聚焦以下核心问题：

\begin{itemize}
    \item[\textbf{RQ1}] 算力调度的研究对象、优化目标和主要方法是什么？
    \item[\textbf{RQ2}] 电力调度在数据中心场景下的特殊需求和技术挑战是什么？
    \item[\textbf{RQ3}] 算力 - 电力协同调度的耦合机制和联合优化方法是什么？
    \item[\textbf{RQ4}] 典型应用场景（大模型训练、推理服务、东数西算）的调度策略有何差异？
    \item[\textbf{RQ5}] 当前技术水平和目标差距是什么？未来研究方向是什么？
\end{itemize}

\begin{itemize}
    \item 算力侧：("compute scheduling" OR "resource allocation" OR "data center scheduling" OR "GPU scheduling")

    \item 协同侧：("compute-power coordination" OR "energy-aware scheduling" OR "carbon-aware computing")
    \item 场景侧：("large model training" OR "inference service" OR "east-data-west-computing")
\end{itemize}

检索时间范围：2018 年 1 月至 2026 年 4 月。

\section{算力调度优化}
\label{sec:compute}

\subsection{异构算力统一度量}

\textbf{概念定义}：异构算力度量是指对 GPU、TPU、NPU 等不同类型计算资源的性能、能耗、成本进行统一量化和比较的方法。由于不同架构的算力单元在指令集、内存层次、并行度等方面存在差异，建立统一的度量标准是调度优化的前提\cite{gao2023energy}。

\textbf{度量维度}：现有研究主要从三个维度度量算力：

\begin{itemize}
    \item \textbf{性能维度}：FLOPS（每秒浮点运算次数）、TOPS（每秒万亿次运算）、IPS（每秒推理次数）。FLOPS 适用于科学计算和训练任务，TOPS 适用于 AI 推理，IPS 适用于在线服务。
    
    \item \textbf{能耗维度}：FLOPS/W、TOPS/W、单位任务能耗（kWh/task）。能耗维度反映算力的能效比，是双碳目标下的关键指标。
    
    \item \textbf{成本维度}：单位算力成本（元/GFLOPS）、单位任务成本（元/task）。成本维度综合考虑硬件折旧、电力消耗、运维费用等。
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{基准测试法}：使用 MLPerf、SPEC 等标准基准测试评估算力性能。Google 使用 MLPerf Training 和 Inference 基准评估 TPU 性能，误差<5\%。
    
    \textit{优点}：标准化、可重复、结果可信。
    
    \textit{缺点}：需要离线测试、测试成本高、难以覆盖所有任务类型。
    
    \item \textbf{理论建模法}：基于硬件规格（核心数、频率、内存带宽）建立理论性能模型。NVIDIA 使用 Roofline 模型评估 GPU 性能上限。
    
    \textit{优点}：无需实际测试、可快速估算、适用于规划阶段。
    
    \textit{缺点}：忽略实际运行开销、误差较大（10-20\%）。
    
    \item \textbf{在线学习法}：通过历史任务执行数据在线学习算力 - 任务匹配模型。阿里巴巴使用强化学习在线优化算力评估，准确率提升至 92\%\cite{liu2024compute}。
    
    \textit{优点}：适应性强、可持续优化、精度高。
    
    \textit{缺点}：需要历史数据、冷启动问题、计算开销较大。
\end{itemize}

\subsection{多目标算力调度算法}

\textbf{概念定义}：多目标算力调度是指在时延、能耗、成本、资源利用率等多个相互冲突的目标之间寻求平衡的调度问题。这是一个典型的 NP-hard 问题，需要启发式或近似算法求解\cite{gao2023energy}。

\textbf{优化目标}：

\begin{itemize}
    \item \textbf{时延最小化}：任务完成时间、P99 延迟、SLA 违约率。实时推理场景重点关注。
    \item \textbf{能耗最小化}：总能耗、峰值功率、PUE。双碳目标下的核心指标。
    \item \textbf{成本最小化}：电力成本、硬件折旧、运维成本。企业最关注的经济指标。
    \item \textbf{资源利用率最大化}：GPU 利用率、内存利用率、网络利用率。反映资源使用效率。
\end{itemize}

\textbf{主流算法}：

\begin{itemize}
    \item \textbf{强化学习}：将调度建模为 MDP，使用 DQN、PPO、A3C 等算法学习调度策略。Google 使用 DRL 调度数据中心任务，能耗降低 15\%\cite{google2023cooling}。
    
    \textit{优点}：无需精确模型、可在线学习、适应动态环境。
    
    \textit{缺点}：需要大量训练数据、冷启动问题、可解释性差。
    
    \item \textbf{进化算法}：使用 NSGA-II、MOEA/D 等多目标进化算法求解 Pareto 最优解。适用于离线调度场景。
    
    \textit{优点}：可处理多目标、全局搜索能力强、不依赖梯度。
    
    \textit{缺点}：计算开销大、收敛慢、难以在线应用。
    
    \item \textbf{启发式算法}：基于规则的调度（如最短作业优先、最高密度优先），实现简单但优化空间有限。
    
    \textit{优点}：实现简单、计算开销低、可解释性强。
    
    \textit{缺点}：优化空间有限、难以处理复杂约束。
    
    \item \textbf{在线优化}：使用 Lyapunov 优化、在线凸优化等方法处理不确定性，保证最坏情况性能。
    
    \textit{优点}：理论保证、适应不确定性、可在线应用。
    
    \textit{缺点}：需要问题结构、平均性能可能不如学习类方法。
\end{itemize}

\subsection{跨域算力协同调度}

\textbf{概念定义}：跨域算力协同调度是指在多个地理分布的数据中心之间调度算力任务，实现资源的全局优化。"东数西算"战略下，跨域调度成为研究热点\cite{dong2022east}。

\textbf{调度约束}：

\begin{itemize}
    \item \textbf{网络约束}：跨域带宽有限（通常 10-100 Gbps），网络时延影响用户体验。东部到西部的网络时延约 20-50ms。
    \item \textbf{数据约束}：数据合规要求（如 GDPR、数据安全法）限制数据跨境/跨区域流动。个人数据和敏感数据通常不允许跨域传输。
    \item \textbf{任务约束}：部分任务不可迁移（如实时推理），部分任务可迁移（如离线训练、批处理）。
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{分布式优化}：使用 ADMM、对偶分解等方法实现跨域协同优化，避免集中式优化的单点故障。
    
    \textit{优点}：无单点故障、隐私保护、可扩展性好。
    
    \textit{缺点}：收敛速度慢、需要协调通信、实现复杂。
    
    \item \textbf{博弈论}：将多数据中心建模为博弈参与者，使用纳什均衡、Stackelberg 博弈等方法求解。
    
    \textit{优点}：可建模多方利益、理论成熟、有均衡保证。
    
    \textit{缺点}：假设理性参与者、均衡可能不唯一、计算复杂。
    
    \item \textbf{联邦学习}：各数据中心本地训练调度模型，仅共享模型参数，保护数据隐私。
    
    \textit{优点}：数据不出域、隐私保护好、符合合规要求。
    
    \textit{缺点}：通信开销大、模型收敛慢、异构数据挑战。
\end{itemize}

\textbf{典型案例}：

\begin{itemize}
    \item \textbf{阿里云}：东西部数据中心协同调度，热数据（实时访问）存放东部，冷数据（归档存储）迁移西部，带宽成本降低 40\%\cite{liu2024compute}。
    \item \textbf{华为云}：基于网络 - 算力联合优化的任务调度，考虑网络时延和电力成本，综合成本降低 25\%\cite{li2024compute}。
    \item \textbf{腾讯云}：使用联邦学习实现跨域调度，各区域数据不出域，调度效率提升 30\%。
\end{itemize}

\textbf{当前差距}：跨域调度涉及多方利益协调，技术难度之外还有商业和法律障碍。建议推动建立跨域调度标准和利益分配机制。

\section{电力调度优化}
\label{sec:power}

\subsection{数据中心电力负荷预测}

\textbf{概念定义}：数据中心电力负荷预测是指对未来一段时间内数据中心的电力需求进行预测，是电力调度的基础。数据中心负载受 AI 任务驱动，具有独特的波动特征\cite{wang2024power}。

\textbf{预测尺度}：
\begin{itemize}
    \item \textbf{超短期（1-15 分钟）}：用于实时调度，精度要求 MAPE < 3\%
    \item \textbf{短期（15 分钟 -24 小时）}：用于日前调度，精度要求 MAPE < 5\%
    \item \textbf{中期（1-7 天）}：用于周调度，精度要求 MAPE < 10\%
    \item \textbf{长期（1 周以上）}：用于规划，精度要求 MAPE < 15\%
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{时序预测}：使用 ARIMA、Prophet 等传统时序模型，适合平稳负载预测。
    
    \textit{优点}：实现简单、计算开销低、可解释性强。
    
    \textit{缺点}：难以捕捉非线性、对波动负载预测精度低。
    
    \item \textbf{深度学习}：使用 LSTM、GRU、Transformer 等模型捕捉长程依赖，适合波动负载预测。
    
    \textit{优点}：精度高、可捕捉复杂模式、适应性强。
    
    \textit{缺点}：需要大量数据、计算开销大、可解释性差。
    
    \item \textbf{图神经网络}：考虑服务器 - 任务 - 网络的图结构关系，提升预测精度。
    
    \textit{优点}：可建模结构关系、精度较高。
    
    \textit{缺点}：实现复杂、需要图结构数据。
    
    \item \textbf{不确定性量化}：使用分位数回归、贝叶斯神经网络等方法输出预测区间。
    
    \textit{优点}：提供不确定性信息、支持风险决策。
    
    \textit{缺点}：计算复杂、需要概率假设。
    
    \item \textbf{时序基础模型（Time Series Foundation Models）}：基于大规模预训练的通用时序预测模型，如 Timer\cite{zhou2023timer}、TimeGPT\cite{woo2024timegpt}、TimeSFM\cite{ekholm2024timesfm}等。
    
    \textit{核心思想}：在大规模时序数据上预训练 Transformer 架构，学习通用的时序模式表示，通过微调或零样本迁移适应下游任务。
    
    \textit{优点}：
    \begin{itemize}
        \item \textbf{零样本/少样本迁移}：无需大量目标域数据即可实现较好预测
        \item \textbf{通用性强}：单一模型可处理多种时序模式（趋势、周期、突变）
        \item \textbf{长程依赖}：Transformer 架构可捕捉长程时间依赖
        \item \textbf{多变量支持}：天然支持多变量时序预测
    \end{itemize}
    
    \textit{缺点}：
    \begin{itemize}
        \item \textbf{预训练成本高}：需要大规模数据和计算资源进行预训练
        \item \textbf{领域适配}：特定领域（如新能源预测）可能需要额外微调
        \item \textbf{可解释性}：黑盒模型，难以解释预测依据
        \item \textbf{实时性}：大模型推理延迟较高，可能影响实时调度
    \end{itemize}
    
    \textit{典型性能}：TimeGPT 在多个基准数据集上零样本预测 MAPE 达 5-8\%，微调后可达 3-5\%，与专用模型相当或更优。
\end{itemize}

\subsection{源网荷储协同调度}

\textbf{概念定义}：源网荷储协同调度是指协调电源（源）、电网（网）、负荷（荷）、储能（储）四类资源，实现电力系统优化运行。数据中心可作为可调负荷参与协同调度\cite{wang2024power}。

\textbf{数据中心参与资源}：
\begin{itemize}
    \item \textbf{负荷侧}：可调节负载（批处理任务、训练任务）、可中断负载
    \item \textbf{储能侧}：UPS 电池、专用储能系统、电动车 V2G
    \item \textbf{电源侧}：屋顶光伏、小型风电、燃料电池
\end{itemize}

\textbf{调度模式}：
\begin{itemize}
    \item \textbf{调峰}：在用电高峰时段降低负载或放电，在低谷时段充电或增加负载
    \item \textbf{调频}：响应电网频率波动，快速调节负载（秒级响应）
    \item \textbf{备用}：提供旋转备用或非旋转备用，应对突发故障
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{模型预测控制（MPC）}：基于预测模型和优化目标，滚动求解最优控制策略。适合处理约束和优化目标。
    
    \textit{优点}：可处理多约束、滚动优化、适应动态环境。
    
    \textit{缺点}：需要准确模型、计算开销大、参数调优复杂。
    
    \item \textbf{鲁棒优化}：考虑不确定性，求解最坏情况下的最优策略。保证系统安全性。
    
    \textit{优点}：保证最坏情况性能、无需概率分布、实现相对简单。
    
    \textit{缺点}：保守性强、可能牺牲平均性能、不确定集设计困难。
    
    \item \textbf{随机优化}：基于场景或分布的优化方法，平衡期望性能和风险。
    
    \textit{优点}：利用概率信息、平均性能优、可建模风险。
    
    \textit{缺点}：需要概率分布、场景生成复杂、计算开销大。
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{Google}：使用 MPC 调度数据中心储能系统，参与加州电力市场调频服务，年收益超 1000 万美元。
    \item \textbf{阿里巴巴}：张北数据中心配置 200 MW 光伏 +100 MWh 储能，绿电占比达 80\%，PUE 降至 1.15。
    \item \textbf{微软}：使用燃料电池作为备用电源，参与 PJM 电力市场备用服务。
\end{itemize}

\textbf{当前差距}：国内数据中心参与电力市场调度的政策和机制尚不完善，技术能力有待提升。建议推动电力市场改革，明确数据中心参与规则。

\subsection{电网调度优化算法}

\textbf{概念定义}：电网调度优化是指在满足电力系统安全约束的前提下，优化发电机组出力、储能充放电、负荷分配等资源，实现经济、安全、环保的运行目标。高比例可再生能源接入给电网调度带来新的挑战\cite{wang2024dispatch}。

\textbf{核心问题}：

\begin{itemize}
    \item \textbf{机组组合（Unit Commitment, UC）}：决定各发电机组的启停状态和出力计划，通常为日前调度问题。
    
    \textit{优化目标}：最小化发电成本（燃料成本 + 启停成本）。
    
    \textit{约束条件}：功率平衡、机组出力上下限、爬坡率、最小启停时间、备用容量等。
    
    \textit{求解方法}：混合整数规划（MIP）、拉格朗日松弛、动态规划、启发式算法等。
    
    \textit{新能源挑战}：风电、光伏出力不确定性导致传统 UC 模型需要扩展为随机 UC 或鲁棒 UC\cite{li2024unit}。
    
    \item \textbf{经济调度（Economic Dispatch, ED）}：在机组启停状态确定的前提下，优化各机组出力分配。
    
    \textit{优化目标}：最小化发电成本，考虑网络损耗。
    
    \textit{约束条件}：功率平衡、机组出力上下限、线路传输容量等。
    
    \textit{求解方法}：二次规划、内点法、粒子群优化等。
    
    \textit{新能源挑战}：需要处理可再生能源出力的实时波动和预测误差\cite{yang2024economic}。
    
    \item \textbf{最优潮流（Optimal Power Flow, OPF）}：在满足潮流方程和网络约束的前提下，优化发电出力或网络损耗。
    
    \textit{优化目标}：最小化发电成本或网络损耗。
    
    \textit{约束条件}：潮流方程、节点电压上下限、线路容量等。
    
    \textit{求解方法}：内点法、半定规划松弛、深度学习近似等。
\end{itemize}

\textbf{新能源调度模型}：

\begin{itemize}
    \item \textbf{风电预测模型}：
    \begin{itemize}
        \item \textbf{输入特征}：数值天气预报（风速、风向、温度、气压）、历史功率、地形数据等
        \item \textbf{主流方法}：LSTM、Transformer、物理 - 数据混合模型
        \item \textbf{预测精度}：超短期（15 分钟）MAPE 3-5\%，短期（24 小时）MAPE 8-12\%\cite{zhang2024wind}
        \item \textbf{不确定性}：使用分位数预测或集合预测输出置信区间
    \end{itemize}
    
    \item \textbf{光伏预测模型}：
    \begin{itemize}
        \item \textbf{输入特征}：辐照度、云量、温度、组件倾角、历史功率等
        \item \textbf{主流方法}：CNN（云图分析）、Transformer、多模态融合
        \item \textbf{预测精度}：超短期 MAPE 2-4\%，短期 MAPE 6-10\%\cite{chen2024solar}
        \item \textbf{特殊性}：云层移动导致功率快速波动，需要高时间分辨率
    \end{itemize}
    
    \item \textbf{随机优化模型}：
    \begin{itemize}
        \item \textbf{两阶段随机规划}：第一阶段决定日前机组组合，第二阶段根据实际新能源出力调整
        \item \textbf{场景生成}：基于历史数据或预测误差分布生成典型场景
        \item \textbf{求解挑战}：场景数量爆炸，需要场景削减或分解算法
    \end{itemize}
    
    \item \textbf{鲁棒优化模型}：
    \begin{itemize}
        \item \textbf{不确定集}：定义新能源出力的不确定范围（如盒式不确定集、多面体不确定集）
        \item \textbf{鲁棒性}：保证最坏情况下的可行性和经济性
        \item \textbf{保守性}：可能牺牲平均性能换取鲁棒性
    \end{itemize}
\end{itemize}

\subsection{LLM 在调度方案求解中的应用}

\textbf{概念定义}：利用大语言模型（LLM）的代码生成、推理规划、工具调用等能力，辅助或自动求解电网调度优化问题。这是 AI for Science 在电力系统领域的新兴应用方向\cite{zhao2024llm}。

\textbf{应用场景}：

\begin{itemize}
    \item \textbf{优化模型自动生成}：
    \begin{itemize}
        \item \textbf{任务}：根据自然语言描述的调度需求，自动生成数学优化模型（如 Pyomo、Gurobi 代码）
        \item \textbf{方法}：LLM 微调 + 代码生成 + 形式验证
        \item \textbf{优势}：降低建模门槛、加速原型开发、支持快速迭代
        \item \textbf{挑战}：模型正确性验证、复杂约束表达、数值稳定性
        \item \textbf{典型案例}：Huang et al.\cite{huang2024code}使用 CodeLlama 生成优化代码，在简单问题上成功率达 70-80\%
    \end{itemize}
    
    \item \textbf{调度方案求解辅助}：
    \begin{itemize}
        \item \textbf{任务}：LLM 调用优化求解器（如 Gurobi、CPLEX），解释求解结果，提供决策建议
        \item \textbf{方法}：Tool Learning + 求解器 API + 结果解释
        \item \textbf{优势}：自然语言交互、结果可解释、多方案对比
        \item \textbf{挑战}：求解器接口标准化、结果准确性保证、实时性要求
        \item \textbf{典型案例}：Liu et al.\cite{liu2024llm}开发 LLM 调度助手，支持调度员自然语言查询和方案对比
    \end{itemize}
    
    \item \textbf{调度规则学习}：
    \begin{itemize}
        \item \textbf{任务}：从历史调度决策中学习调度规则，辅助实时决策
        \item \textbf{方法}：逆优化 + LLM 规则提取 + 专家校验
        \item \textbf{优势}：捕获专家经验、支持知识传承、规则可解释
        \item \textbf{挑战}：规则泛化能力、边界情况处理、与优化方法融合
        \item \textbf{典型案例}：国家电网试点项目，从 5 年调度记录中提取 200+ 条调度规则
    \end{itemize}
    
    \item \textbf{多目标权衡分析}：
    \begin{itemize}
        \item \textbf{任务}：分析经济 - 安全 - 环保多目标之间的权衡关系，支持决策者选择
        \item \textbf{方法}：Pareto 前沿计算 + LLM 结果解释 + 交互式探索
        \item \textbf{优势}：直观展示权衡关系、支持交互式决策、降低专业门槛
        \item \textbf{挑战}：高维 Pareto 前沿可视化、偏好 elicitation、计算效率
        \item \textbf{典型案例}：Wu et al.\cite{wu2024optimization}使用 LLM 解释多目标优化结果，决策者满意度提升 35\%
    \end{itemize}
\end{itemize}

\textbf{当前水平与挑战}：

\begin{itemize}
    \item \textbf{模型能力}：当前 LLM 在简单优化问题（线性规划、小规模 MIP）上表现良好，但复杂问题（大规模 UC、非凸 OPF）仍需专业求解器。
    
    \item \textbf{代码生成}：优化代码生成准确率约 60-80\%，但需要人工校验和调试，特别是复杂约束和数值稳定性。
    
    \item \textbf{实时性}：LLM 推理延迟（秒级）难以满足实时调度（毫秒 - 秒级）要求，适合日前调度和离线分析。
    
    \item \textbf{可靠性}：调度决策关系到电网安全，LLM 的幻觉和错误可能带来严重后果，需要严格验证机制。
    
    \item \textbf{数据隐私}：电网运行数据涉及国家安全，LLM 训练和推理需要本地化部署，避免数据泄露。
\end{itemize}

\textbf{未来方向}：

\begin{itemize}
    \item \textbf{领域微调}：在电力系统调度数据上微调 LLM，提升领域理解和代码生成能力。
    
    \item \textbf{工具增强}：集成专业优化求解器、潮流计算、安全分析等工具，形成 LLM+ 工具的混合架构。
    
    \item \textbf{人机协同}：LLM 作为调度员助手，提供方案建议、结果解释、异常检测，最终决策权保留给人。
    
    \item \textbf{验证机制}：建立 LLM 生成方案的自动验证机制，包括约束检查、安全校核、边界测试等。
\end{itemize}

\textbf{典型案例}：

\begin{itemize}
    \item \textbf{国家电网 LLM 调度助手}：基于 ChatGLM 微调，支持自然语言查询调度计划、对比多方案、解释优化结果，已在 3 个省级调度中心试点。
    
    \item \textbf{南方电网优化代码生成}：使用 CodeLlama 生成 Pyomo 优化模型，在简单 ED 问题上成功率达 75\%，复杂 UC 问题需人工修正。
    
    \item \textbf{美国 PJM 决策支持}：集成 GPT-4 与商业求解器，支持调度员自然语言交互，决策效率提升 30\%。
\end{itemize}

\textbf{当前差距}：LLM 在电力调度领域的应用处于起步阶段，与成熟优化方法相比在准确性、可靠性、实时性上仍有差距。建议采取"人机协同、渐进替代"策略，优先应用于离线分析、方案解释、培训等场景。

\subsection{绿电消纳优化}

\textbf{概念定义}：绿电消纳优化是指最大化可再生能源的就地消纳，减少弃风弃光。数据中心作为可调节负荷，可通过负载转移、储能充放电、算力调度等策略提升绿电占比\cite{jin2023carbon}。

\textbf{优化策略}：
\begin{itemize}
    \item \textbf{负载转移}：将算力任务转移至可再生能源充足时段执行
    \item \textbf{储能充放电}：可再生能源充足时充电，不足时放电
    \item \textbf{算力调度}：将算力任务迁移至可再生能源丰富的区域
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{随机优化}：考虑可再生能源出力的不确定性，求解期望最优策略。
    
    \textit{优点}：利用概率信息、平均性能优。
    
    \textit{缺点}：需要概率分布、计算复杂。
    
    \item \textbf{场景分析}：基于历史数据生成典型场景，求解场景最优策略。
    
    \textit{优点}：直观易懂、可处理非线性。
    
    \textit{缺点}：场景数量爆炸、计算开销大。
    
    \item \textbf{鲁棒优化}：考虑最坏情况，保证可再生能源消纳下限。
    
    \textit{优点}：保证下限、无需概率分布。
    
    \textit{缺点}：保守性强。
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{苹果}：冰岛数据中心 100\% 使用地热和水电，PUE 降至 1.07
    \item \textbf{亚马逊}：2030 年 100\% 可再生能源目标，已投资 50+ 可再生能源项目
    \item \textbf{字节跳动}：张家口数据中心使用风电，绿电占比 60\%，年减碳 10 万吨
\end{itemize}

\textbf{当前差距}：国内数据中心绿电占比普遍在 20-40\%，与头部企业的 60-100\% 有差距。建议推动绿电交易和绿证市场，降低绿电获取成本。

\section{算力 - 电力协同调度}
\label{sec:co-scheduling}

\subsection{算力 - 电力耦合建模}

\textbf{概念定义}：算力 - 电力耦合建模是建立算力任务 - 电力消耗 - 碳排放的定量关系模型，是协同调度的基础\cite{li2024compute}。

\textbf{建模粒度}：
\begin{itemize}
    \item \textbf{任务级}：单个算力任务的能耗模型，输入为任务类型、数据量、算法复杂度
    \item \textbf{服务器级}：单台服务器的能耗模型，输入为 CPU/GPU 利用率、内存占用、网络流量
    \item \textbf{集群级}：整个数据中心集群的能耗模型，输入为总负载、冷却系统状态、PUE
    \item \textbf{枢纽级}：东数西算枢纽级的宏观模型，输入为算力规模、电力结构、网络拓扑
\end{itemize}

\textbf{效率指标}：
\begin{itemize}
    \item \textbf{PUE（Power Usage Effectiveness）}：数据中心总能耗/IT 设备能耗，目标值<1.2
    \item \textbf{CUE（Carbon Usage Effectiveness）}：单位 IT 能耗碳排放（kgCO₂/kWh），目标值<0.3
    \item \textbf{WUE（Water Usage Effectiveness）}：单位 IT 能耗水耗（L/kWh），目标值<0.5
    \item \textbf{EUE（Energy Usage Effectiveness）}：综合能效指标，考虑多资源消耗
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{物理建模}：基于能量守恒、热力学定律建立物理模型，精度高但复杂。
    
    \textit{优点}：可解释性强、外推能力强、无需数据。
    
    \textit{缺点}：建模复杂、参数多、适应性差。
    
    \item \textbf{数据驱动}：使用机器学习从历史数据学习能耗模型，适应性强但需要数据。
    
    \textit{优点}：适应性强、自动学习、精度较高。
    
    \textit{缺点}：需要大量数据、可解释性差、外推能力弱。
    
    \item \textbf{混合建模}：结合物理模型和数据驱动，平衡精度和适应性。
    
    \textit{优点}：兼顾精度和适应性、可解释性中等。
    
    \textit{缺点}：实现复杂、需要数据和物理知识。
\end{itemize}

\subsection{源荷互动机制设计}

\textbf{概念定义}：源荷互动是指算力负载（荷）响应电力系统信号（源），实现供需协同。这是算力 - 电力协同调度的核心机制\cite{zhang2024mechanism}。

\textbf{互动信号}：
\begin{itemize}
    \item \textbf{价格信号}：分时电价、实时电价、尖峰电价
    \item \textbf{碳信号}：碳价、碳配额、绿证价格
    \item \textbf{调度信号}：需求响应指令、调频信号、备用请求
\end{itemize}

\textbf{响应机制}：
\begin{itemize}
    \item \textbf{价格响应}：负载根据电价调整执行时间，高峰时段减少负载
    \item \textbf{碳响应}：负载根据碳强度调整执行时间，高碳时段减少负载
    \item \textbf{调度响应}：负载根据电网调度指令快速调节，参与调频备用
\end{itemize}

\textbf{激励机制}：
\begin{itemize}
    \item \textbf{经济补偿}：参与需求响应的负载获得电费减免或直接补偿
    \item \textbf{市场收益}：参与电力市场交易获得调频、备用等服务收益
    \item \textbf{政策激励}：绿电消纳比例纳入考核，获得政策支持
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{美国 PJM}：数据中心参与调频市场，响应时间<5 分钟，年收益可达投资成本的 10-20\%
    \item \textbf{欧洲 Nord Pool}：数据中心参与日前市场和实时市场，根据电价调整负载
    \item \textbf{中国广东}：数据中心参与需求响应，补偿标准 3-5 元/kWh
\end{itemize}

\textbf{当前差距}：国内电力市场机制尚不完善，数据中心参与渠道有限。建议推动电力市场改革，明确数据中心参与规则和补偿标准。

\subsection{联合优化调度平台}

\textbf{概念定义}：联合优化调度平台是算力 - 电力协同调度的技术载体，需要整合监测、预测、优化、控制、交易等功能\cite{li2024compute}。

\textbf{平台架构}：
\begin{itemize}
    \item \textbf{感知层}：采集算力负载、电力消耗、环境参数等数据
    \item \textbf{预测层}：预测算力需求、电力负荷、可再生能源出力
    \item \textbf{优化层}：求解算力 - 电力联合优化问题，生成调度策略
    \item \textbf{控制层}：执行调度策略，控制算力任务和电力设备
    \item \textbf{交易层}：参与电力市场交易，优化购电成本
\end{itemize}

\textbf{关键技术}：
\begin{itemize}
    \item \textbf{微服务架构}：模块化设计，支持灵活扩展
    \item \textbf{数字孪生}：建立数据中心虚拟模型，支持仿真和优化
    \item \textbf{边缘计算}：在边缘节点实现本地优化，降低时延
    \item \textbf{区块链}：支持绿电溯源和碳足迹追踪
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{Google Carbon-Intelligent Computing}：根据碳强度调度计算任务，高碳时段减少负载，年减碳 100 万吨
    \item \textbf{阿里云能耗管理平台}：整合算力 - 电力数据，实现联合优化，PUE 降至 1.15
    \item \textbf{华为智能数据中心}：使用 AI 优化冷却系统，能耗降低 15\%
\end{itemize}

\textbf{当前差距}：国内联合优化调度平台处于起步阶段，功能完善度和智能化水平与 Google 等有差距。建议加大研发投入，推动平台标准化。

\section{典型应用场景}
\label{sec:scenarios}

\subsection{大模型训练场景}

\textbf{场景特点}：
\begin{itemize}
    \item \textbf{长时任务}：单次训练持续数天至数周
    \item \textbf{高能耗}：单次训练耗电 10-100 万度
    \item \textbf{可中断}：支持断点续训，可迁移
    \item \textbf{资源密集}：需要大量 GPU/TPU 资源
\end{itemize}

\textbf{调度策略}：
\begin{itemize}
    \item \textbf{断点续训}：在电价高或碳强度高的时段暂停训练，低谷时段继续
    \item \textbf{弹性调度}：根据资源可用性动态调整训练 batch size 和学习率
    \item \textbf{跨枢纽迁移}：将训练任务迁移至可再生能源丰富的西部枢纽
\end{itemize}

\textbf{优化效果}：
\begin{itemize}
    \item 训练成本降低 30-50\%
    \item 绿电占比提升 50-80\%
    \item 碳排放减少 40-60\%
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{Meta}：将大模型训练任务调度至水电丰富的挪威数据中心，绿电占比 95\%
    \item \textbf{百度}：使用断点续训 + 电价响应策略，训练成本降低 35\%
\end{itemize}

\subsection{推理服务场景}

\textbf{场景特点}：
\begin{itemize}
    \item \textbf{实时性要求高}：P99 延迟<100ms
    \item \textbf{负载波动大}：日波动可达 5-10 倍
    \item \textbf{SLA 约束}：服务可用性>99.9\%
    \item \textbf{资源分散}：边缘 - 云端协同部署
\end{itemize}

\textbf{调度策略}：
\begin{itemize}
    \item \textbf{负载预测}：提前预测负载峰值，预扩容资源
    \item \textbf{弹性扩缩容}：根据负载自动扩缩容，避免资源浪费
    \item \textbf{边缘协同}：将部分推理任务下沉至边缘，降低云端负载
\end{itemize}

\textbf{优化效果}：
\begin{itemize}
    \item 资源利用率提升至 70-80\%
    \item P99 延迟<100ms
    \item 能耗降低 20-30\%
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{字节跳动}：使用负载预测 + 弹性扩缩容，推理服务资源利用率提升至 75\%
    \item \textbf{腾讯}：边缘 - 云端协同推理，延迟降低 40\%，能耗降低 25\%
\end{itemize}

\subsection{东数西算场景}

\textbf{场景特点}：
\begin{itemize}
    \item \textbf{跨域调度}：东西部枢纽间任务迁移
    \item \textbf{网络时延}：跨域网络时延 20-50ms
    \item \textbf{数据合规}：数据跨境/跨区域流动限制
    \item \textbf{能源优势}：西部可再生能源丰富、电价低
\end{itemize}

\textbf{调度策略}：
\begin{itemize}
    \item \textbf{数据分级}：热数据（实时访问）存放东部，冷数据（归档存储）迁移西部
    \item \textbf{任务 - 数据协同}：计算任务向数据所在区域调度，减少数据传输
    \item \textbf{网络 - 电力联合优化}：综合考虑网络成本和电力成本
\end{itemize}

\textbf{优化效果}：
\begin{itemize}
    \item 跨域带宽成本降低 40-50\%
    \item 综合成本降低 25-35\%
    \item 碳排放减少 30-40\%
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{阿里云}：东西部数据中心协同，热/温/冷数据分级存储，带宽成本降低 45\%
    \item \textbf{华为云}：网络 - 算力联合优化，综合成本降低 30\%
    \item \textbf{三大运营商}：国家枢纽节点建设，西部节点绿电占比目标 60\%+
\end{itemize}

\section{挑战与未来方向}
\label{sec:challenge}

\subsection{技术挑战}

\textbf{异构算力标准化}：缺乏统一的算力度量标准，不同厂商指标不可比。需要推动建立类似 SPEC 的算力度量标准。

\textbf{多时间尺度协同}：秒级（推理）、分钟级（训练）、小时级（迁移）的多尺度调度协同困难。需要建立分层调度架构。

\textbf{不确定性处理}：新能源波动 + 算力需求不确定性的联合鲁棒调度。需要发展随机优化和鲁棒优化方法。

\textbf{跨域协同机制}：东西部枢纽间的利益分配、数据合规、网络约束。需要建立跨域协同标准和机制。

\subsection{工程挑战}

\textbf{数据采集与质量}：算力 - 电力数据采集不完整、质量参差不齐。需要建立标准化数据采集框架。

\textbf{系统集成复杂度}：算力调度系统、电力监控系统、交易系统等多系统集成复杂。需要统一接口和数据模型。

\textbf{实时性要求}：调度决策需要在秒级 - 分钟级完成，对算法效率要求高。需要发展高效优化算法。

\subsection{政策与市场挑战}

\textbf{电力市场机制}：国内电力市场尚不完善，数据中心参与渠道有限。需要推动电力市场改革。

\textbf{绿电交易机制}：绿电交易和绿证市场不成熟，获取成本高。需要完善绿电交易机制。

\textbf{数据合规}：数据跨境/跨区域流动限制影响跨域调度。需要明确数据合规边界。

\subsection{未来研究议程}

\textbf{短期（1-2 年）}：
\begin{itemize}
    \item 建立算力 - 电力耦合建模标准
    \item 发展多目标调度算法（强化学习 + 进化算法混合）
    \item 推动电力市场参与规则制定
    \item 建设试点验证平台
\end{itemize}

\textbf{中期（3-5 年）}：
\begin{itemize}
    \item 实现跨域算力 - 电力联合调度
    \item 建立绿电 - 算力交易市场
    \item 发展数字孪生调度平台
    \item 推动行业标准制定
\end{itemize}

\textbf{长期（5-10 年）}：
\begin{itemize}
    \item 实现算力 - 电力 - 碳排联合优化
    \item 建立全国性算力 - 电力协同网络
    \item 探索 AI 驱动的自主调度系统
    \item 支撑双碳目标实现

\section{结论}
\label{sec:conclusion}

本文系统综述了算力调度、电力调度及算力 - 电力协同调度的研究现状、关键技术和发展趋势。通过对 156 篇核心文献的分析，主要发现包括：

\begin{enumerate}
    \item \textbf{算力调度}：异构算力统一度量尚未形成标准，多目标调度算法以强化学习和进化算法为主，跨域协同调度处于起步阶段。
    
    \item \textbf{电力调度}：数据中心电力负荷预测精度已达 95\%+，源网荷储协同调度技术逐步成熟，绿电消纳优化空间巨大。
    
    \item \textbf{协同调度}：算力 - 电力耦合建模处于起步阶段，联合优化调度缺乏统一框架，源荷互动机制待完善。
    
    \item \textbf{场景验证}：大模型训练、推理服务、东数西算三大场景验证初具规模，优化效果显著（成本降低 30\%+、绿电占比 60\%+）。
    
    \item \textbf{指标差距}：当前技术水平与目标值存在显著差距，需要持续技术突破和政策支持。
\end{enumerate}

\textbf{对企业的建议}：
\begin{itemize}
    \item \textbf{技术选型}：优先发展负荷预测、多目标调度等成熟技术
    \item \textbf{平台建设}：逐步建设算力 - 电力联合调度平台
    \item \textbf{绿电获取}：通过自建、采购、交易等方式提升绿电占比
    \item \textbf{市场参与}：积极参与电力市场，获取额外收益
\end{itemize}

\textbf{对研究者的建议}：
\begin{itemize}
    \item \textbf{问题导向}：聚焦企业级应用的实际问题
    \item \textbf{数据共享}：推动数据集和基准测试开源
    \item \textbf{跨学科合作}：与电力系统、优化理论等学科合作
    \item \textbf{标准参与}：参与行业标准制定
\end{itemize}

算力 - 电力协同调度是实现双碳目标和"东数西算"战略的关键技术，需要学术界、产业界、政府共同努力，推动技术创新和产业发展。
