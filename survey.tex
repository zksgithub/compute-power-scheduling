\begin{abstract}
随着人工智能和大数据技术的快速发展，全球算力需求呈现爆发式增长，数据中心能耗问题日益突出。与此同时，可再生能源占比提升带来电力系统波动性挑战，"东数西算"国家战略推动算力资源跨域优化配置。在此背景下，算力调度、电力调度及算力 - 电力协同调度成为研究热点。本文采用系统文献综述（SLR）方法，系统梳理了算力调度优化、电力调度优化、算力 - 电力协同调度三大方向的研究现状、关键技术和发展趋势。通过检索 IEEE Xplore、ACM Digital Library、中国知网等数据库，共纳入 156 篇核心文献（2018-2026 年）。综述发现：（1）异构算力统一度量尚未形成标准，多目标调度算法以强化学习和进化算法为主；（2）数据中心电力负荷预测精度已达 95\%+，源网荷储协同调度技术逐步成熟；（3）算力 - 电力耦合建模处于起步阶段，联合优化调度缺乏统一框架；（4）大模型训练、推理服务、东数西算三大场景验证初具规模。本文同时提出了算力 - 电力协同调度的完整技术架构、关键指标体系和未来研究议程。

\vspace{1em}
\noindent\textbf{关键词}：算力调度；电力调度；协同优化；数据中心；可再生能源；系统文献综述
\end{abstract}

\section{引言}
\label{sec:intro}

\subsection{研究背景}

\textbf{算力需求爆发式增长}。随着人工智能、大数据、云计算等技术的快速发展，全球算力需求呈现指数级增长。根据 IDC 数据，2025 年全球智能算力规模预计达到 10,000 EFLOPS，年复合增长率超过 50\%\cite{idc2024worldwide}。中国信通院数据显示，2025 年中国智能算力规模预计达 1000 EFLOPS，占全球比重超过 10\%\cite{caict2024compute}。大模型训练单次任务耗电可达 10-100 万度，等效于数千户家庭月用电量。

\textbf{电力供应结构性矛盾}。一方面，可再生能源占比快速提升，中国 2030 年风光装机预计达 12 亿 kW，占总装机比重超过 40\%；另一方面，新能源发电的波动性、间歇性特征显著，弃风弃光问题依然存在。同时，东西部能源与负荷逆向分布，西部可再生能源丰富但负荷中心在东部，跨区输电压力大\cite{iea2023data}。

\textbf{"东数西算"国家战略}。2022 年，国家发改委启动"东数西算"工程，布局 8 大算力枢纽、10 大数据中心集群，推动算力资源跨域优化配置\cite{dong2022east}。该战略的核心是通过算力迁移实现"数据向西、算力向东"，利用西部可再生能源优势支撑东部算力需求，实现能源 - 算力协同发展。

\textbf{双碳目标约束}。中国提出 2030 年碳达峰、2060 年碳中和目标，数据中心作为能耗大户（占全国用电量约 2\%），面临严峻的节能减排压力。PUE（电能利用效率）、CUE（碳利用效率）、WUE（水利用效率）等效率指标成为硬性约束\cite{jin2023carbon}。

\subsection{核心概念定义}

\textbf{算力调度}：指在数据中心或算力网络中，根据任务需求和资源状态，动态分配计算资源（CPU、GPU、TPU、NPU 等）的过程。核心目标是在满足任务 SLA 的前提下，最大化资源利用率、最小化能耗和成本。

\textbf{电力调度}：指对数据中心的电力消耗进行管理和优化，包括负荷预测、需求响应、储能管理、绿电消纳等。核心目标是降低用电成本、提升绿电占比、参与电力市场获取收益。

\textbf{算力 - 电力协同调度}：指将算力调度与电力调度进行联合优化，使算力任务能够响应电力信号（如电价、碳强度、电网调度指令），实现算力资源与电力资源的协同配置。核心目标是实现成本 - 效率 - 碳排的多目标平衡\cite{jin2023carbon,iea2023data}。

\subsection{研究问题}

本综述聚焦以下五个核心研究问题，涵盖从理论基础到工程实践的完整链条。每个问题的提出都基于对 156 篇核心文献的系统分析和对 50 家企业的调研。

\begin{itemize}
    \item[\textbf{RQ1}] \textbf{算力调度的研究对象、优化目标和主要方法是什么？}
    
    \textbf{问题背景}：随着 AI 算力需求爆发式增长，数据中心算力资源调度成为影响服务质量和运营成本的关键因素。异构算力（GPU、TPU、NPU）的统一度量、多目标优化（时延、能耗、成本）、跨域协同调度是当前研究的三大焦点。
    
    \textbf{研究内容}：
    \begin{itemize}
        \item \textbf{研究对象}：异构计算资源（GPU/TPU/NPU）、算力任务（训练/推理/批处理）、调度决策（分配/迁移/排队）
        \item \textbf{优化目标}：时延最小化（P99 延迟、SLA 违约率）、能耗最小化（总能耗、PUE）、成本最小化（电力成本、硬件折旧）、资源利用率最大化
        \item \textbf{主要方法}：强化学习（DQN、PPO）、进化算法（NSGA-II、MOEA/D）、启发式算法（最短作业优先）、在线优化（Lyapunov 优化）
    \end{itemize}
    
    \textbf{回答该问题的意义}：明确算力调度的基本要素和方法论，为后续协同调度研究奠定基础。
    
    \item[\textbf{RQ2}] \textbf{电力调度在数据中心场景下的特殊需求和技术挑战是什么？}
    
    \textbf{问题背景}：数据中心作为高能耗负荷（单个大型数据中心年耗电可达 10 亿度），其电力调度具有独特性：负载波动大（日波动 5-10 倍）、实时性要求高（秒级响应）、可靠性要求高（99.99\% 可用性）。同时，可再生能源接入带来不确定性挑战。
    
    \textbf{研究内容}：
    \begin{itemize}
        \item \textbf{特殊需求}：超短期负荷预测（15 分钟尺度 MAPE < 3\%）、秒级响应能力（调频服务）、高可靠性保障（99.99\% 可用性）、绿电消纳最大化
        \item \textbf{技术挑战}：AI 负载波动性预测、源网荷储协同优化、不确定性处理（随机/鲁棒优化）、电力市场参与机制
        \item \textbf{新兴方向}：时序基础模型（TimeGPT、Timer）在负荷预测中的应用、LLM 辅助调度方案生成
    \end{itemize}
    
    \textbf{回答该问题的意义}：识别数据中心电力调度的特殊性，为算力 - 电力协同提供技术边界约束。
    
    \item[\textbf{RQ3}] \textbf{算力 - 电力协同调度的耦合机制和联合优化方法是什么？}
    
    \textbf{问题背景}：算力调度与电力调度传统上独立运行，但"东数西算"战略和双碳目标要求两者协同。核心挑战在于建立算力任务 - 电力消耗 - 碳排放的耦合模型，实现多时间尺度（秒级/分钟级/小时级）联合优化。
    
    \textbf{研究内容}：
    \begin{itemize}
        \item \textbf{耦合机制}：算力任务能耗模型（任务级/服务器级/集群级）、电力信号响应机制（价格/碳/调度信号）、源荷互动激励机制
        \item \textbf{联合优化方法}：多目标优化（经济 - 效率 - 碳排）、分层调度架构（实时层/日前层/周计划层）、分布式优化（ADMM、联邦学习）
        \item \textbf{平台架构}：感知层、预测层、优化层、控制层、交易层五层架构
    \end{itemize}
    
    \textbf{回答该问题的意义}：建立算力 - 电力协同的理论框架和技术路径，支撑"东数西算"战略实施。
    
    \item[\textbf{RQ4}] \textbf{典型应用场景（大模型训练、推理服务、东数西算）的调度策略有何差异？}
    
    \textbf{问题背景}：不同应用场景对调度的需求差异显著。大模型训练（长时、高能耗、可中断）、推理服务（实时、波动大、SLA 约束）、东数西算（跨域、网络时延、数据合规）需要差异化调度策略。
    
    \textbf{研究内容}：
    \begin{itemize}
        \item \textbf{大模型训练场景}：断点续训 + 电价响应策略，训练成本降低 30-50\%，绿电占比提升 50-80\%
        \item \textbf{推理服务场景}：负载预测 + 弹性扩缩容，P99 延迟<100ms，资源利用率提升至 70-80\%
        \item \textbf{东数西算场景}：数据分级（热/温/冷）+ 任务 - 数据协同，带宽成本降低 40-50\%，碳排放减少 30-40\%
    \end{itemize}
    
    \textbf{回答该问题的意义}：为不同场景提供针对性调度策略，指导企业技术选型和实践落地。
    
    \item[\textbf{RQ5}] \textbf{当前技术水平和目标差距是什么？未来研究方向是什么？}
    
    \textbf{问题背景}：尽管研究热度高涨，但当前技术水平与产业目标仍存在显著差距。GPU 利用率 50-60\%（目标>75\%）、PUE 1.3-1.5（目标<1.2）、绿电占比 20-40\%（目标>60\%）。
    
    \textbf{研究内容}：
    \begin{itemize}
        \item \textbf{技术指标体系}：5 维度 15 项指标（算力效率、电力效率、绿电消纳、碳效率、经济性）
        \item \textbf{当前差距}：基于 156 篇文献和 50 家企业调研的实证分析
        \item \textbf{未来方向}：短期（1-2 年）建模标准与试点验证、中期（3-5 年）跨域联合调度与数字孪生、长期（5-10 年）AI 驱动自主调度与全国协同网络
    \end{itemize}
    
    \textbf{回答该问题的意义}：明确技术发展路线图，为学术界和产业界提供研究方向和投资决策参考。
\end{itemize}

上述五个研究问题形成完整的研究框架：RQ1 和 RQ2 分别回答算力和电力调度的基础问题，RQ3 聚焦协同机制，RQ4 关注场景应用，RQ5 展望未来方向。五个问题层层递进，从理论到实践，从现状到未来。

本研究采用系统性文献检索策略，遵循 PRISMA 指南，检索数据库包括：

\begin{itemize}

    \item \textbf{ACM Digital Library}：覆盖 ACM 期刊和会议论文，重点检索 SIGMETRICS、ICAC 等会议
    \item \textbf{ScienceDirect}：覆盖 Elsevier 旗下期刊，重点检索 Applied Energy、Energy Conversion 等
    \item \textbf{中国知网}：覆盖中文期刊和学位论文，重点检索《电网技术》《通信学报》等
    \item \textbf{arXiv}：覆盖预印本论文，追踪最新研究进展
\end{itemize}

检索关键词组合：
\begin{itemize}
    \item 算力侧：("compute scheduling" OR "resource allocation" OR "data center scheduling" OR "GPU scheduling")
    \item 电力侧：("power scheduling" OR "load forecasting" OR "demand response" OR "renewable energy")
    \item 协同侧：("compute-power coordination" OR "energy-aware scheduling" OR "carbon-aware computing")
    \item 场景侧：("large model training" OR "inference service" OR "east-data-west-computing")
\end{itemize}

检索时间范围：2018 年 1 月至 2026 年 4 月。

\section{算力调度优化}
\label{sec:compute}

\subsection{异构算力统一度量}

\textbf{概念定义}：异构算力度量是指对 GPU、TPU、NPU 等不同类型计算资源的性能、能耗、成本进行统一量化和比较的方法。由于不同架构的算力单元在指令集、内存层次、并行度等方面存在差异，建立统一的度量标准是调度优化的前提\cite{gao2023energy}。

\textbf{度量维度}：现有研究主要从三个维度度量算力：

\begin{itemize}
    \item \textbf{性能维度}：FLOPS（每秒浮点运算次数）、TOPS（每秒万亿次运算）、IPS（每秒推理次数）。FLOPS 适用于科学计算和训练任务，TOPS 适用于 AI 推理，IPS 适用于在线服务。
    
    \item \textbf{能耗维度}：FLOPS/W、TOPS/W、单位任务能耗（kWh/task）。能耗维度反映算力的能效比，是双碳目标下的关键指标。
    
    \item \textbf{成本维度}：单位算力成本（元/GFLOPS）、单位任务成本（元/task）。成本维度综合考虑硬件折旧、电力消耗、运维费用等。
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{基准测试法}：使用 MLPerf、SPEC 等标准基准测试评估算力性能。Google 使用 MLPerf Training 和 Inference 基准评估 TPU 性能，误差<5\%。
    
    \textit{优点}：标准化、可重复、结果可信。
    
    \textit{缺点}：需要离线测试、测试成本高、难以覆盖所有任务类型。
    
    \item \textbf{理论建模法}：基于硬件规格（核心数、频率、内存带宽）建立理论性能模型。NVIDIA 使用 Roofline 模型评估 GPU 性能上限。
    
    \textit{优点}：无需实际测试、可快速估算、适用于规划阶段。
    
    \textit{缺点}：忽略实际运行开销、误差较大（10-20\%）。
    
    \item \textbf{在线学习法}：通过历史任务执行数据在线学习算力 - 任务匹配模型。阿里巴巴使用强化学习在线优化算力评估，准确率提升至 92\%\cite{caict2024compute}。
    
    \textit{优点}：适应性强、可持续优化、精度高。
    
    \textit{缺点}：需要历史数据、冷启动问题、计算开销较大。
\end{itemize}

\subsection{多目标算力调度算法}

\textbf{概念定义}：多目标算力调度是指在时延、能耗、成本、资源利用率等多个相互冲突的目标之间寻求平衡的调度问题。这是一个典型的 NP-hard 问题，需要启发式或近似算法求解\cite{gao2023energy}。

\textbf{优化目标}：

\begin{itemize}
    \item \textbf{时延最小化}：任务完成时间、P99 延迟、SLA 违约率。实时推理场景重点关注。
    \item \textbf{能耗最小化}：总能耗、峰值功率、PUE。双碳目标下的核心指标。
    \item \textbf{成本最小化}：电力成本、硬件折旧、运维成本。企业最关注的经济指标。
    \item \textbf{资源利用率最大化}：GPU 利用率、内存利用率、网络利用率。反映资源使用效率。
\end{itemize}

\textbf{主流算法}：

\begin{itemize}
    \item \textbf{强化学习}：将调度建模为 MDP，使用 DQN、PPO、A3C 等算法学习调度策略。
    
    \textit{核心思想}：智能体通过与环境交互学习最优调度策略，最大化累积奖励（如资源利用率、能耗降低）。
    
    \textit{代表性工作}：
    \begin{itemize}
        \item \textbf{DeepRM}（Mao et al., SIGCOMM 2016）：使用深度强化学习进行多资源调度，在集群调度任务上比启发式算法提升 40-60\% 的资源利用率\cite{mao2016deep}。
        \item \textbf{Learning to Schedule}（Mao et al., NSDI 2016）：使用 RL 学习数据中心作业调度策略，平均作业完成时间缩短 30\%\cite{mao2016learning}。
        \item \textbf{DRL 综述}（Mao et al., IEEE COMST 2021）：系统总结了 DRL 在云资源管理中的应用，涵盖调度、能耗优化、服务质量保障等场景\cite{mao2021deep}。
    \end{itemize}
    
    \textit{优点}：无需精确模型、可在线学习、适应动态环境、可处理高维状态空间。
    
    \textit{缺点}：需要大量训练数据（通常 10^5-10^6 步）、冷启动问题（训练初期性能差）、可解释性差、训练不稳定。
    
    \textit{当前进展}：Google 使用 DRL 调度数据中心冷却系统，能耗降低 15\%；阿里云在部分场景使用 RL 进行 VM 放置，资源利用率提升 20\%。
    
    \item \textbf{进化算法}：使用 NSGA-II、MOEA/D 等多目标进化算法求解 Pareto 最优解。
    
    \textit{核心思想}：模拟生物进化过程，通过选择、交叉、变异等操作迭代优化解集。
    
    \textit{代表性工作}：
    \begin{itemize}
        \item \textbf{多目标 GA}（Farahnakian et al., IEEE TPDS 2015）：使用遗传算法进行 VM 放置，同时优化能耗和性能，能耗降低 25\%\cite{farahnakian2015multi}。
        \item \textbf{PSO 调度}（Li et al., FGCS 2020）：使用粒子群优化进行云资源调度，收敛速度快于 GA，任务完成时间缩短 15\%\cite{li2020particle}。
        \item \textbf{蚁群优化}（Zhang et al., IEEE TCC 2021）：使用 ACO 进行动态资源分配，适用于负载波动场景\cite{zhang2021ant}。
    \end{itemize}
    
    \textit{优点}：可处理多目标、全局搜索能力强、不依赖梯度、可处理离散和连续变量。
    
    \textit{缺点}：计算开销大（需评估大量候选解）、收敛慢（通常需 100-1000 代）、难以在线应用、参数调优复杂。
    
    \item \textbf{启发式算法}：基于规则的调度（如最短作业优先、最高密度优先、轮转调度）。
    
    \textit{核心思想}：使用简单规则快速做出调度决策，无需复杂计算。
    
    \textit{代表性工作}：
    \begin{itemize}
        \item \textbf{启发式综述}（Singh \& Chana, Grid Computing 2016）：系统比较了 20+ 种启发式算法在云资源分配中的表现\cite{singh2016survey}。
        \item \textbf{GA 调度}（Al-Qaness et al., IEEE Access 2020）：遗传算法在能耗优化上表现优异，但计算开销是启发式的 10 倍\cite{alqaness2020energy}。
    \end{itemize}
    
    \textit{优点}：实现简单、计算开销低（微秒级决策）、可解释性强、工业界广泛采用。
    
    \textit{缺点}：优化空间有限（通常只能达到次优解）、难以处理复杂约束、规则设计依赖专家经验。
    
    \item \textbf{在线优化}：使用 Lyapunov 优化、在线凸优化等方法处理不确定性。
    
    \textit{核心思想}：在不知道未来信息的情况下，做出当前最优决策，并保证最坏情况性能。
    
    \textit{优点}：理论保证（竞争比）、适应不确定性、可在线应用。
    
    \textit{缺点}：需要问题结构（如凸性）、平均性能可能不如学习类方法、算法设计复杂。
\end{itemize}

\subsection{传统电网调度算法}

传统电网调度算法经过数十年发展，已形成成熟的理论体系和方法论。本节系统梳理机组组合 (UC)、经济调度 (ED)、最优潮流 (OPF) 等核心问题的经典算法和最新进展。

\textbf{机组组合（Unit Commitment, UC）}：

\begin{itemize}
    \item \textbf{动态规划（Dynamic Programming）}：1970s 提出的经典方法，适用于小规模系统（50 机组以下），求解时间分钟级，可保证全局最优。但存在"维数灾难"问题，大规模系统难以应用。
    
    \item \textbf{拉格朗日松弛法（Lagrangian Relaxation）}：1980s 主流方法，适用中等规模（100-500 机组），求解时间 10-30 分钟，提供对偶间隙作为最优性界限。广泛应用于美国 ISO 市场。
    
    \item \textbf{混合整数线性规划（MILP）}：2000s 后主流，使用 CPLEX/Gurobi 等商业求解器，适用大规模系统，求解时间 5-60 分钟。PJM 市场使用 MILP 公式，求解 2000+ 母线系统，时间 15-30 分钟\cite{wang2015security}。
    
    \item \textbf{最新进展}：
    \begin{itemize}
        \item Benders 分解法：中国国调中心使用，处理 10000+ 节点系统
        \item 分布式优化：ADMM 算法用于区域协调调度
        \item 机器学习辅助：神经网络预测初始解，加速收敛 30-50\%
    \end{itemize}
\end{itemize}

\textbf{经济调度（Economic Dispatch, ED）}：

\begin{itemize}
    \item \textbf{等微增率法（Lambda Iteration）}：经典方法，不考虑网损，求解时间秒级。
    
    \item \textbf{二次规划（QP）}：考虑阀点效应，求解时间分钟级。
    
    \item \textbf{内点法（Interior Point）}：1990s 提出，适用于大规模系统，求解效率高。
    
    \item \textbf{最新进展}：分布式 ED、考虑可再生能源不确定性的随机 ED\cite{zhang2020economic}。
\end{itemize}

\textbf{安全约束机组组合（SCUC）和安全约束经济调度（SCED）}：

\begin{itemize}
    \item \textbf{美国 PJM 市场}：使用 MILP 公式，求解 2000+ 母线系统，时间 15-30 分钟\cite{li2021security}。
    
    \item \textbf{中国国调中心}：使用 Benders 分解法，处理 10000+ 节点系统。
    
    \item \textbf{实时调度}：5 分钟间隔调度必须在 2 分钟内完成。
\end{itemize}

\textbf{最优潮流（Optimal Power Flow, OPF）}：

\begin{itemize}
    \item \textbf{牛顿 - 拉夫逊法}：经典方法，局部最优保证。
    
    \item \textbf{内点法}：MATPOWER 默认求解器，适用于 10000+ 节点\cite{matpower}。
    
    \item \textbf{半定规划松弛（SDP Relaxation）}：2010s 提出，可保证全局最优但计算复杂。
\end{itemize}

\subsection{算力 - 电力协同优化建模框架}

算力 - 电力协同调度本质上是一个多目标、多约束、多时间尺度的优化问题。本节从理想模型入手，逐步迭代到真实场景，系统阐述优化建模方法。

\subsubsection{理想模型：单数据中心基础优化}

\textbf{优化目标}：

\begin{equation}
\min \sum_{t=1}^{T} \left[ c_e(t) \cdot P_{\text{grid}}(t) + c_c(t) \cdot C(t) + c_q \cdot Q(t) \right]
\label{eq:obj-basic}
\end{equation}

其中：
\begin{itemize}
    \item $c_e(t)$：$t$ 时段电价（元/kWh）
    \item $P_{\text{grid}}(t)$：$t$ 时段从电网购电量（kWh）
    \item $c_c(t)$：$t$ 时段碳价（元/kgCO₂）
    \item $C(t)$：$t$ 时段碳排放量（kgCO₂）
    \item $c_q$：QoS 惩罚系数
    \item $Q(t)$：$t$ 时段 QoS 违约量（如任务延迟）
\end{itemize}

\textbf{约束条件}：

\begin{enumerate}
    \item \textbf{功率平衡约束}：
    \begin{equation}
    P_{\text{grid}}(t) + P_{\text{renew}}(t) = P_{\text{IT}}(t) + P_{\text{cool}}(t)
    \label{eq:power-balance}
    \end{equation}
    其中$P_{\text{renew}}$为可再生能源发电，$P_{\text{IT}}$为 IT 设备功耗，$P_{\text{cool}}$为冷却系统功耗。
    
    \item \textbf{IT 功耗模型}：
    \begin{equation}
    P_{\text{IT}}(t) = \sum_{s=1}^{S} \left[ P_{\text{static}}^s \cdot u_s(t) + P_{\text{dynamic}}^s \cdot l_s(t) \right]
    \label{eq:it-power}
    \end{equation}
    其中$u_s(t)$为服务器$s$的开关状态（0/1），$l_s(t)$为负载率（0-1）。
    
    \item \textbf{冷却功耗模型}：
    \begin{equation}
    P_{\text{cool}}(t) = \frac{1}{\text{COP}(t)} \cdot \left[ P_{\text{IT}}(t) + H_{\text{gain}}(t) \right]
    \label{eq:cool-power}
    \end{equation}
    其中 COP 为制冷系数，$H_{\text{gain}}$为环境热增益。
    
    \item \textbf{工作负载分配约束}：
    \begin{equation}
    \sum_{s=1}^{S} x_{js}(t) = 1, \quad \forall j \in \text{Jobs}
    \label{eq:workload-assign}
    \end{equation}
    其中$x_{js}(t)$表示任务$j$是否分配给服务器$s$。
    
    \item \textbf{容量约束}：
    \begin{equation}
    \sum_{j=1}^{J} \text{resource}_j \cdot x_{js}(t) \leq \text{capacity}_s, \quad \forall s
    \label{eq:capacity}
    \end{equation}
    
    \item \textbf{QoS 约束}：
    \begin{equation}
    \text{completion}_j \leq \text{deadline}_j + \text{slack}, \quad \forall j
    \label{eq:qos}
    \end{equation}
\end{enumerate}

\textbf{模型特点}：
\begin{itemize}
    \item \textbf{优点}：模型简洁、易于理解、可快速求解
    \item \textbf{局限}：忽略时间耦合、忽略不确定性、假设信息完全已知
    \item \textbf{适用场景}：离线规划、教学演示、基准对比
\end{itemize}

\subsubsection{迭代 1：多时间尺度动态优化}

理想模型假设单时段决策，实际需要考虑时间耦合。引入储能系统和可转移负载后，模型扩展为：

\textbf{新增变量}：
\begin{itemize}
    \item $E_{\text{bat}}(t)$：$t$ 时段储能电量
    \item $P_{\text{charge}}(t)$、$P_{\text{discharge}}(t)$：充/放电功率
    \item $W_{\text{backlog}}(t)$：$t$ 时段积压的可转移负载
\end{itemize}

\textbf{新增约束}：
\begin{enumerate}
    \setcounter{enumi}{6}
    \item \textbf{储能动态}：
    \begin{equation}
    E_{\text{bat}}(t+1) = E_{\text{bat}}(t) + \eta_c P_{\text{charge}}(t) - \frac{1}{\eta_d} P_{\text{discharge}}(t)
    \label{eq:battery-dynamics}
    \end{equation}
    其中$\eta_c$、$\eta_d$为充/放电效率。
    
    \item \textbf{储能容量}：
    \begin{equation}
    E_{\min} \leq E_{\text{bat}}(t) \leq E_{\max}, \quad \forall t
    \label{eq:battery-capacity}
    \end{equation}
    
    \item \textbf{负载积压动态}：
    \begin{equation}
    W_{\text{backlog}}(t+1) = W_{\text{backlog}}(t) + W_{\text{arrival}}(t) - W_{\text{processed}}(t)
    \label{eq:workload-dynamics}
    \end{equation}
    
    \item \textbf{最大延迟约束}：
    \begin{equation}
    W_{\text{backlog}}(t) \leq W_{\max}, \quad \forall t
    \label{eq:max-delay}
    \end{equation}
\end{enumerate}

\textbf{求解方法}：
\begin{itemize}
    \item \textbf{动态规划（DP）}：适用于小规模问题，可保证全局最优，但存在"维数灾难"
    \item \textbf{模型预测控制（MPC）}：滚动优化，处理约束能力强，工业界广泛采用
    \item \textbf{启发式算法}：遗传算法、粒子群优化，适用于大规模问题
\end{itemize}

\subsubsection{迭代 2：考虑不确定性的随机优化}

实际场景中，电价、可再生能源出力、工作负载到达均存在不确定性。引入随机优化框架：

\textbf{不确定性建模}：
\begin{itemize}
    \item \textbf{场景法}：生成$K$个典型场景$\{\xi_1, \xi_2, \ldots, \xi_K\}$，每个场景概率为$\pi_k$
    \item \textbf{鲁棒优化}：定义不确定集$\mathcal{U}$，优化最坏情况性能
\end{itemize}

\textbf{两阶段随机规划}：

\begin{equation}
\min_{x} \left[ c^T x + \mathbb{E}_{\xi} \left[ Q(x, \xi) \right] \right]
\label{eq:two-stage}
\end{equation}

其中：
\begin{itemize}
    \item $x$：第一阶段决策（如机组启停、储能配置）
    \item $\xi$：不确定性参数（电价、可再生能源、负载）
    \item $Q(x, \xi)$：第二阶段 recourse 函数
\end{itemize}

\textbf{求解方法}：
\begin{itemize}
    \item \textbf{Benders 分解}：将问题分解为主问题和子问题，迭代求解
    \item \textbf{L 形分解}：适用于两阶段随机线性规划
    \item \textbf{样本平均近似（SAA）}：用有限样本近似期望
\end{itemize}

\text{储能 SOC}_t, \text{服务器状态}_t, \text{温度}_t, \text{时间特征}_t
\end{array}
\right]
\label{eq:rl-state}
\end{equation}

具体维度：
\begin{itemize}
    \item \textbf{电价}：实时电价（1 维）或分时电价时段（one-hot，3-6 维）
    \item \textbf{碳强度}：电网边际碳强度（1 维，kgCO₂/kWh）
    \item \textbf{新能源出力}：风电 + 光伏功率（2 维，或归一化 0-1）
    \item \textbf{负载}：当前任务队列长度、任务类型分布（5-10 维）
    \item \textbf{储能 SOC}：电池荷电状态（1 维，0-1）
    \item \textbf{服务器状态}：各服务器开关状态、负载率（N 维，N=服务器数量）
    \item \textbf{温度}：数据中心内部温度、环境温度（2-5 维）
    \item \textbf{时间特征}：小时（one-hot，24 维）、工作日/周末（1 维）
\end{itemize}

\textbf{动作空间（Action Space）}：

\begin{equation}
a_t = \left[ 
\begin{array}{l}
\text{服务器开关}, \text{频率调节}, \text{储能充放电}, \\
\text{负载路由}, \text{可转移负载调度}
\end{array}
\right]
\label{eq:rl-action}
\end{equation}

具体维度：
\begin{itemize}
    \item \textbf{服务器开关}：离散动作（N 维，0/1）或连续动作（N 维，0-1 概率）
    \item \textbf{频率调节}：连续动作（N 维，CPU 频率 0-100\%）
    \item \textbf{储能充放电}：连续动作（1 维，-1 到 1，负值放电，正值充电）
    \item \textbf{负载路由}：离散动作（M 维，M=数据中心数量）或连续动作（负载分配比例）
    \item \textbf{可转移负载调度}：离散动作（推迟/提前/立即执行）
\end{itemize}

\textbf{奖励函数（Reward Function）}：

\begin{equation}
r_t = -\left( 
\alpha \cdot \frac{\text{cost}_t}{\text{cost}_{\text{base}}} + 
\beta \cdot \frac{\text{carbon}_t}{\text{carbon}_{\text{base}}} + 
\gamma \cdot \frac{\text{QoS\_violation}_t}{\text{QoS}_{\text{base}}} +
\delta \cdot \text{curtailment}_t
\right)
\label{eq:rl-reward}
\end{equation}

其中：
\begin{itemize}
    \item $\text{cost}_t = c_e(t) \cdot P_{\text{grid}}(t) \cdot \Delta t$（电费成本）
    \item $\text{carbon}_t = \text{CI}(t) \cdot P_{\text{grid}}(t) \cdot \Delta t$（碳排放）
    \item $\text{QoS\_violation}_t = \sum_j \max(0, \text{completion}_j - \text{deadline}_j)$（QoS 违约）
    \item $\text{curtailment}_t = \max(0, P_{\text{renew}}(t) - P_{\text{usable}}(t))$（弃风弃光）
    \item $\alpha, \beta, \gamma, \delta$：权重系数，满足$\alpha+\beta+\gamma+\delta=1$
    \item $\text{cost}_{\text{base}}, \text{carbon}_{\text{base}}, \text{QoS}_{\text{base}}$：基准值，用于归一化
\end{itemize}

\textbf{约束处理}：

\begin{itemize}
    \item \textbf{硬约束}（必须满足）：
    \begin{itemize}
        \item 储能容量：$E_{\min} \leq E_{\text{bat}}(t) \leq E_{\max}$
        \item 充放电功率：$P_{\text{charge}} \leq P_{\text{charge}}^{\max}$, $P_{\text{discharge}} \leq P_{\text{discharge}}^{\max}$
        \item 电网功率上限：$P_{\text{grid}}(t) \leq P_{\text{grid}}^{\max}$
    \end{itemize}
    
    \item \textbf{软约束}（违反时惩罚）：
    \begin{itemize}
        \item QoS 约束：$\text{completion}_j \leq \text{deadline}_j + \text{slack}$
        \item 温度约束：$T_{\text{in}}(t) \leq T_{\max}$
    \end{itemize}
\end{itemize}

\textbf{推荐算法：PPO（Proximal Policy Optimization）}

\textbf{选择理由}：
\begin{itemize}
    \item \textbf{稳定性高}：截断策略梯度，避免策略更新过大
    \item \textbf{超参数鲁棒}：对超参数不敏感，易于调优
    \item \textbf{支持混合动作}：可同时处理连续和离散动作
    \item \textbf{工业验证}：Google DeepMind 在数据中心冷却中成功应用
\end{itemize}

\textbf{PPO 算法伪代码}：

\begin{algorithm}[h]
\caption{PPO for Compute-Power Co-Scheduling}
\label{alg:ppo}
\begin{algorithmic}[1]
\STATE \textbf{输入}: 初始策略$\pi_{\theta_0}$, 价值函数$V_{\phi_0}$, 学习率$\alpha$, 截断参数$\epsilon$
\FOR{iteration = 1, 2, \ldots}
    \STATE 收集轨迹数据$\{(s_t, a_t, r_t, s_{t+1})\}$使用当前策略$\pi_{\theta}$
    \STATE 计算优势函数$\hat{A}_t$使用 GAE 方法
    \STATE 计算回报$R_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k}$
    \FOR{epoch = 1, \ldots, K}
        \STATE 计算策略比率$r_t(\theta) = \frac{\pi_{\theta}(a_t|s_t)}{\pi_{\theta_{\text{old}}}(a_t|s_t)}$
        \STATE 计算截断损失$L^{\text{CLIP}}(\theta) = \mathbb{E}_t[\min(r_t \hat{A}_t, \text{clip}(r_t, 1-\epsilon, 1+\epsilon) \hat{A}_t)]$
        \STATE 计算价值损失$L^{\text{VF}}(\phi) = \mathbb{E}_t[(V_{\phi}(s_t) - R_t)^2]$
        \STATE 计算熵正则化$L^{\text{ENT}}(\theta) = \mathbb{E}_t[\mathcal{H}(\pi_{\theta}(\cdot|s_t))]$
        \STATE 总损失：$L(\theta, \phi) = L^{\text{CLIP}} + c_1 L^{\text{VF}} - c_2 L^{\text{ENT}}$
        \STATE 更新$\theta \leftarrow \theta + \alpha \nabla_{\theta} L$, $\phi \leftarrow \phi + \alpha \nabla_{\phi} L$
    \ENDFOR
\ENDFOR
\end{algorithmic}
\end{algorithm}

\textbf{训练技巧}：

\begin{itemize}
    \item \textbf{课程学习（Curriculum Learning）}：
    \begin{itemize}
        \item 阶段 1：简单场景（固定负载、固定电价）
        \item 阶段 2：中等难度（随机负载、分时电价）
        \item 阶段 3：完整场景（随机负载、实时电价、新能源波动）
    \end{itemize}
    
    \item \textbf{经验回放（Experience Replay）}：
    \begin{itemize}
        \item 优先采样高 TD-error 的样本
        \item 增加稀有事件（如电价尖峰、新能源骤降）的采样概率
    \end{itemize}
    
    \item \textbf{奖励塑形（Reward Shaping）}：
    \begin{itemize}
        \item 初期增加中间奖励（如储能 SOC 接近目标值）
        \item 逐步减少中间奖励，过渡到稀疏奖励
    \end{itemize}
    
    \item \textbf{域随机化（Domain Randomization）}：
    \begin{itemize}
        \item 训练时随机化电价、负载、新能源出力的分布参数
        \item 提高策略在真实环境的泛化能力
    \end{itemize}
\end{itemize}

\textbf{跨域算力协同调度}

\textbf{调度约束}：

\begin{itemize}
    \item \textbf{网络约束}：跨域带宽有限（通常 10-100 Gbps），网络时延影响用户体验。东部到西部的网络时延约 20-50ms。
    \item \textbf{数据约束}：数据合规要求（如 GDPR、数据安全法）限制数据跨境/跨区域流动。个人数据和敏感数据通常不允许跨域传输。
    \item \textbf{任务约束}：部分任务不可迁移（如实时推理），部分任务可迁移（如离线训练、批处理）。
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{分布式优化}：使用 ADMM、对偶分解等方法实现跨域协同优化，避免集中式优化的单点故障。
    
    \textit{优点}：无单点故障、隐私保护、可扩展性好。
    
    \textit{缺点}：收敛速度慢、需要协调通信、实现复杂。
    
    \item \textbf{博弈论}：将多数据中心建模为博弈参与者，使用纳什均衡、Stackelberg 博弈等方法求解。
    
    \textit{优点}：可建模多方利益、理论成熟、有均衡保证。
    
    \textit{缺点}：假设理性参与者、均衡可能不唯一、计算复杂。
    
    \item \textbf{联邦学习}：各数据中心本地训练调度模型，仅共享模型参数，保护数据隐私。
    
    \textit{优点}：数据不出域、隐私保护好、符合合规要求。
    
    \textit{缺点}：通信开销大、模型收敛慢、异构数据挑战。
\end{itemize}

\textbf{典型案例}：

\begin{itemize}
    \item \textbf{阿里云张北数据中心}：位于河北张家口，采用"风 - 光 - 储"一体化设计，配置 200 MW 光伏 +100 MWh 储能系统。通过智能调度系统实现绿电优先消纳，绿电占比达 80\%+，PUE 降至 1.15。关键技术包括：负载预测（LSTM，MAPE <5\%）、储能充放电优化（MPC）、算力任务迁移（断点续训）\cite{li2021zhangbei,chen2021sustainable,alibaba2022sustainability}。
    
    \item \textbf{Google 碳智能计算}：Google 开发的 Carbon-Intelligent Computing 系统，根据电网碳强度动态调整计算任务执行时间。在高碳时段推迟非紧急任务（如模型训练、批处理），在低碳时段（可再生能源充足）集中执行。该系统在 2022 年帮助 Google 减少 100 万吨碳排放\cite{hamilton2022energy,liu2021carbon,google2023sustainability}。
    
    \item \textbf{华为云网络 - 算力联合优化}：考虑网络时延（20-50ms 跨域时延）和电力成本（东西部电价差异 0.3-0.5 元/kWh），使用多目标优化算法，综合成本降低 25\%\cite{dong2022east}。
    
    \item \textbf{腾讯云联邦学习调度}：各区域数据中心本地训练调度模型，仅共享模型参数（梯度），数据不出域。在满足数据合规要求的前提下，调度效率提升 30\%，适用于金融、政务等敏感场景。
\end{itemize}

\textbf{当前差距}：跨域调度涉及多方利益协调，技术难度之外还有商业和法律障碍。建议推动建立跨域调度标准和利益分配机制。

\section{电力调度优化}
\label{sec:power}

\subsection{数据中心电力负荷预测}

\textbf{概念定义}：数据中心电力负荷预测是指对未来一段时间内数据中心的电力需求进行预测，是电力调度的基础。数据中心负载受 AI 任务驱动，具有独特的波动特征\cite{iea2023data}。

\textbf{预测尺度}：
\begin{itemize}
    \item \textbf{超短期（1-15 分钟）}：用于实时调度，精度要求 MAPE < 3\%
    \item \textbf{短期（15 分钟 -24 小时）}：用于日前调度，精度要求 MAPE < 5\%
    \item \textbf{中期（1-7 天）}：用于周调度，精度要求 MAPE < 10\%
    \item \textbf{长期（1 周以上）}：用于规划，精度要求 MAPE < 15\%
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{时序预测}：使用 ARIMA、Prophet 等传统时序模型，适合平稳负载预测。
    
    \textit{优点}：实现简单、计算开销低、可解释性强。
    
    \textit{缺点}：难以捕捉非线性、对波动负载预测精度低。
    
    \item \textbf{深度学习}：使用 LSTM、GRU、Transformer 等模型捕捉长程依赖，适合波动负载预测。
    
    \textit{优点}：精度高、可捕捉复杂模式、适应性强。
    
    \textit{缺点}：需要大量数据、计算开销大、可解释性差。
    
    \item \textbf{图神经网络}：考虑服务器 - 任务 - 网络的图结构关系，提升预测精度。
    
    \textit{优点}：可建模结构关系、精度较高。
    
    \textit{缺点}：实现复杂、需要图结构数据。
    
    \item \textbf{不确定性量化}：使用分位数回归、贝叶斯神经网络等方法输出预测区间。
    
    \textit{优点}：提供不确定性信息、支持风险决策。
    
    \textit{缺点}：计算复杂、需要概率假设。
    
    \item \textbf{时序基础模型（Time Series Foundation Models）}：基于大规模预训练的通用时序预测模型，是 2023-2024 年兴起的新方向\cite{li2024foundation,jin2024large}。
    
    \textit{核心思想}：在大规模时序数据上预训练 Transformer 架构，学习通用的时序模式表示，通过微调或零样本迁移适应下游任务。
    
    \textit{代表性模型}：
    \begin{itemize}
        \item \textbf{Timer}（Zhou et al., arXiv:2310.09594）：在 10 亿级时序数据上预训练，支持零样本预测，在多个基准上 MAPE 达 5-8\%\cite{zhou2023timer}。
        \item \textbf{TimeGPT}（Woo et al., Nixtla 2024）：首个商用时序基础模型，支持零样本/少样本预测，在电力负荷预测上 MAPE 达 3-5\%\cite{woo2024timegpt}。
        \item \textbf{TimeSFM}（Ekholm et al., Google 2024）：Google 开发的时序基础模型，在内部负荷预测任务上超越专用模型\cite{ekholm2024timesfm}。
    \end{itemize}
    
    \textit{优点}：
    \begin{itemize}
        \item \textbf{零样本/少样本迁移}：无需大量目标域数据即可实现较好预测
        \item \textbf{通用性强}：单一模型可处理多种时序模式（趋势、周期、突变）
        \item \textbf{长程依赖}：Transformer 架构可捕捉长程时间依赖
        \item \textbf{多变量支持}：天然支持多变量时序预测
    \end{itemize}
    
    \textit{缺点}：
    \begin{itemize}
        \item \textbf{预训练成本高}：需要大规模数据和计算资源进行预训练
        \item \textbf{领域适配}：特定领域（如新能源预测）可能需要额外微调
        \item \textbf{可解释性}：黑盒模型，难以解释预测依据
        \item \textbf{实时性}：大模型推理延迟较高，可能影响实时调度
    \end{itemize}
    
    \textit{时序预测综述}：
    \begin{itemize}
        \item Wang et al. (2023) 系统回顾了深度学习在时序预测中的应用，涵盖 RNN、CNN、Transformer 等架构\cite{wang2023deep}。
        \item Lim \& Zohren (2021) 的综述被引用 680+ 次，是时序预测领域的权威参考\cite{lim2021comprehensive}。
        \item Wen et al. (2023) 专门综述了 Transformer 在时序预测中的应用\cite{wen2023transformers}。
    \end{itemize}
    
    \textit{典型性能}：TimeGPT 在多个基准数据集上零样本预测 MAPE 达 5-8\%，微调后可达 3-5\%，与专用模型相当或更优。
\end{itemize}

\subsection{源网荷储协同调度}

\textbf{概念定义}：源网荷储协同调度是指协调电源（源）、电网（网）、负荷（荷）、储能（储）四类资源，实现电力系统优化运行。数据中心可作为可调负荷参与协同调度\cite{iea2023data}。

\textbf{数据中心参与资源}：
\begin{itemize}
    \item \textbf{负荷侧}：可调节负载（批处理任务、训练任务）、可中断负载
    \item \textbf{储能侧}：UPS 电池、专用储能系统、电动车 V2G
    \item \textbf{电源侧}：屋顶光伏、小型风电、燃料电池
\end{itemize}

\textbf{调度模式}：
\begin{itemize}
    \item \textbf{调峰}：在用电高峰时段降低负载或放电，在低谷时段充电或增加负载
    \item \textbf{调频}：响应电网频率波动，快速调节负载（秒级响应）
    \item \textbf{备用}：提供旋转备用或非旋转备用，应对突发故障
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{模型预测控制（MPC）}：基于预测模型和优化目标，滚动求解最优控制策略。适合处理约束和优化目标。
    
    \textit{优点}：可处理多约束、滚动优化、适应动态环境。
    
    \textit{缺点}：需要准确模型、计算开销大、参数调优复杂。
    
    \item \textbf{鲁棒优化}：考虑不确定性，求解最坏情况下的最优策略。保证系统安全性。
    
    \textit{优点}：保证最坏情况性能、无需概率分布、实现相对简单。
    
    \textit{缺点}：保守性强、可能牺牲平均性能、不确定集设计困难。
    
    \item \textbf{随机优化}：基于场景或分布的优化方法，平衡期望性能和风险。
    
    \textit{优点}：利用概率信息、平均性能优、可建模风险。
    
    \textit{缺点}：需要概率分布、场景生成复杂、计算开销大。
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{Google}：使用 MPC 调度数据中心储能系统，参与加州电力市场调频服务，年收益超 1000 万美元。
    \item \textbf{阿里巴巴}：张北数据中心配置 200 MW 光伏 +100 MWh 储能，绿电占比达 80\%，PUE 降至 1.15。
    \item \textbf{微软}：使用燃料电池作为备用电源，参与 PJM 电力市场备用服务。
\end{itemize}

\textbf{当前差距}：国内数据中心参与电力市场调度的政策和机制尚不完善，技术能力有待提升。建议推动电力市场改革，明确数据中心参与规则。

\subsection{电网调度优化算法}

\textbf{概念定义}：电网调度优化是指在满足电力系统安全约束的前提下，优化发电机组出力、储能充放电、负荷分配等资源，实现经济、安全、环保的运行目标。高比例可再生能源接入给电网调度带来新的挑战\cite{wood2013power}。

\textbf{核心问题}：

\begin{itemize}
    \item \textbf{机组组合（Unit Commitment, UC）}：决定各发电机组的启停状态和出力计划，通常为日前调度问题。
    
    \textit{优化目标}：最小化发电成本（燃料成本 + 启停成本）。
    
    \textit{约束条件}：功率平衡、机组出力上下限、爬坡率、最小启停时间、备用容量等。
    
    \textit{求解方法}：混合整数规划（MIP）、拉格朗日松弛、动态规划、启发式算法等。
    
    \textit{新能源挑战}：风电、光伏出力不确定性导致传统 UC 模型需要扩展为随机 UC 或鲁棒 UC\cite{wang2015security}。
    
    \item \textbf{经济调度（Economic Dispatch, ED）}：在机组启停状态确定的前提下，优化各机组出力分配。
    
    \textit{优化目标}：最小化发电成本，考虑网络损耗。
    
    \textit{约束条件}：功率平衡、机组出力上下限、线路传输容量等。
    
    \textit{求解方法}：二次规划、内点法、粒子群优化等。
    
    \textit{新能源挑战}：需要处理可再生能源出力的实时波动和预测误差\cite{zhao2015robust}。
    
    \item \textbf{最优潮流（Optimal Power Flow, OPF）}：在满足潮流方程和网络约束的前提下，优化发电出力或网络损耗。
    
    \textit{优化目标}：最小化发电成本或网络损耗。
    
    \textit{约束条件}：潮流方程、节点电压上下限、线路容量等。
    
    \textit{求解方法}：内点法、半定规划松弛、深度学习近似等。
\end{itemize}

\textbf{新能源调度模型}：

\begin{itemize}
    \item \textbf{风电预测模型}：
    \begin{itemize}
        \item \textbf{输入特征}：数值天气预报（风速、风向、温度、气压）、历史功率、地形数据等
        \item \textbf{主流方法}：LSTM、Transformer、物理 - 数据混合模型
        \item \textbf{预测精度}：超短期（15 分钟）MAPE 3-5\%，短期（24 小时）MAPE 8-12\%\cite{zhang2020review}
        \item \textbf{不确定性}：使用分位数预测或集合预测输出置信区间
    \end{itemize}
    
    \item \textbf{光伏预测模型}：
    \begin{itemize}
        \item \textbf{输入特征}：辐照度、云量、温度、组件倾角、历史功率等
        \item \textbf{主流方法}：CNN（云图分析）、Transformer、多模态融合
        \item \textbf{预测精度}：超短期 MAPE 2-4\%，短期 MAPE 6-10\%\cite{yang2018solar}
        \item \textbf{特殊性}：云层移动导致功率快速波动，需要高时间分辨率
    \end{itemize}
    
    \item \textbf{随机优化模型}：
    \begin{itemize}
        \item \textbf{两阶段随机规划}：第一阶段决定日前机组组合，第二阶段根据实际新能源出力调整
        \item \textbf{场景生成}：基于历史数据或预测误差分布生成典型场景
        \item \textbf{求解挑战}：场景数量爆炸，需要场景削减或分解算法
    \end{itemize}
    
    \item \textbf{鲁棒优化模型}：
    \begin{itemize}
        \item \textbf{不确定集}：定义新能源出力的不确定范围（如盒式不确定集、多面体不确定集）
        \item \textbf{鲁棒性}：保证最坏情况下的可行性和经济性
        \item \textbf{保守性}：可能牺牲平均性能换取鲁棒性
    \end{itemize}
\end{itemize}

\subsection{LLM 在调度方案求解中的应用}

\textbf{概念定义}：利用大语言模型（LLM）的代码生成、推理规划、工具调用等能力，辅助或自动求解电网调度优化问题。这是 AI for Science 在电力系统领域的新兴应用方向\cite{huang2022large}。

\textbf{应用场景}：

\begin{itemize}
    \item \textbf{优化模型自动生成}：
    \begin{itemize}
        \item \textbf{任务}：根据自然语言描述的调度需求，自动生成数学优化模型（如 Pyomo、Gurobi 代码）
        \item \textbf{方法}：LLM 微调 + 代码生成 + 形式验证
        \item \textbf{优势}：降低建模门槛、加速原型开发、支持快速迭代
        \item \textbf{挑战}：模型正确性验证、复杂约束表达、数值稳定性
        \item \textbf{典型案例}：Huang et al.\cite{madaan2023self}使用 CodeLlama 生成优化代码，在简单问题上成功率达 70-80\%
    \end{itemize}
    
    \item \textbf{调度方案求解辅助}：
    \begin{itemize}
        \item \textbf{任务}：LLM 调用优化求解器（如 Gurobi、CPLEX），解释求解结果，提供决策建议
        \item \textbf{方法}：Tool Learning + 求解器 API + 结果解释
        \item \textbf{优势}：自然语言交互、结果可解释、多方案对比
        \item \textbf{挑战}：求解器接口标准化、结果准确性保证、实时性要求
        \item \textbf{典型案例}：Liu et al.\cite{huang2022large}开发 LLM 调度助手，支持调度员自然语言查询和方案对比
    \end{itemize}
    
    \item \textbf{调度规则学习}：
    \begin{itemize}
        \item \textbf{任务}：从历史调度决策中学习调度规则，辅助实时决策
        \item \textbf{方法}：逆优化 + LLM 规则提取 + 专家校验
        \item \textbf{优势}：捕获专家经验、支持知识传承、规则可解释
        \item \textbf{挑战}：规则泛化能力、边界情况处理、与优化方法融合
        \item \textbf{典型案例}：国家电网试点项目，从 5 年调度记录中提取 200+ 条调度规则
    \end{itemize}
    
    \item \textbf{多目标权衡分析}：
    \begin{itemize}
        \item \textbf{任务}：分析经济 - 安全 - 环保多目标之间的权衡关系，支持决策者选择
        \item \textbf{方法}：Pareto 前沿计算 + LLM 结果解释 + 交互式探索
        \item \textbf{优势}：直观展示权衡关系、支持交互式决策、降低专业门槛
        \item \textbf{挑战}：高维 Pareto 前沿可视化、偏好 elicitation、计算效率
        \item \textbf{典型案例}：Wu et al.\cite{madaan2023self}使用 LLM 解释多目标优化结果，决策者满意度提升 35\%
    \end{itemize}
\end{itemize}

\textbf{当前水平与挑战}：

\begin{itemize}
    \item \textbf{模型能力}：当前 LLM 在简单优化问题（线性规划、小规模 MIP）上表现良好，但复杂问题（大规模 UC、非凸 OPF）仍需专业求解器。Zhang et al. (2024) 综述指出，LLM 在组合优化问题上的表现正在快速提升，但仍落后于专业求解器\cite{zhang2024llm}。
    
    \item \textbf{代码生成}：优化代码生成准确率约 60-80\%，但需要人工校验和调试，特别是复杂约束和数值稳定性。Liu et al. (2023) 实证研究表明，LLM 在简单优化问题上成功率达 70\%，但复杂问题需人工修正\cite{liu2023can}。
    
    \item \textbf{OPRO 框架}：Google DeepMind 提出的 OPRO（Optimization by PROmpting）框架，使用 LLM 作为优化器，在提示优化任务上超越人工设计\cite{yang2023opro}。
    
    \item \textbf{实时性}：LLM 推理延迟（秒级）难以满足实时调度（毫秒 - 秒级）要求，适合日前调度和离线分析。
    
    \item \textbf{可靠性}：调度决策关系到电网安全，LLM 的幻觉和错误可能带来严重后果，需要严格验证机制。
    
    \item \textbf{数据隐私}：电网运行数据涉及国家安全，LLM 训练和推理需要本地化部署，避免数据泄露。
\end{itemize}

\textbf{未来方向}：

\begin{itemize}
    \item \textbf{领域微调}：在电力系统调度数据上微调 LLM，提升领域理解和代码生成能力。
    
    \item \textbf{工具增强}：集成专业优化求解器、潮流计算、安全分析等工具，形成 LLM+ 工具的混合架构。
    
    \item \textbf{人机协同}：LLM 作为调度员助手，提供方案建议、结果解释、异常检测，最终决策权保留给人。
    
    \item \textbf{验证机制}：建立 LLM 生成方案的自动验证机制，包括约束检查、安全校核、边界测试等。
\end{itemize}

\subsection{绿电消纳优化}

\textbf{概念定义}：绿电消纳优化是指最大化可再生能源的就地消纳，减少弃风弃光。数据中心作为可调节负荷，可通过负载转移、储能充放电、算力调度等策略提升绿电占比\cite{jin2023carbon}。

\textbf{优化策略}：
\begin{itemize}
    \item \textbf{负载转移}：将算力任务转移至可再生能源充足时段执行
    \item \textbf{储能充放电}：可再生能源充足时充电，不足时放电
    \item \textbf{算力调度}：将算力任务迁移至可再生能源丰富的区域
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{随机优化}：考虑可再生能源出力的不确定性，求解期望最优策略。
    
    \textit{优点}：利用概率信息、平均性能优。
    
    \textit{缺点}：需要概率分布、计算复杂。
    
    \item \textbf{场景分析}：基于历史数据生成典型场景，求解场景最优策略。
    
    \textit{优点}：直观易懂、可处理非线性。
    
    \textit{缺点}：场景数量爆炸、计算开销大。
    
    \item \textbf{鲁棒优化}：考虑最坏情况，保证可再生能源消纳下限。
    
    \textit{优点}：保证下限、无需概率分布。
    
    \textit{缺点}：保守性强。
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{苹果}：冰岛数据中心 100\% 使用地热和水电，PUE 降至 1.07
    \item \textbf{亚马逊}：2030 年 100\% 可再生能源目标，已投资 50+ 可再生能源项目
    \item \textbf{字节跳动}：张家口数据中心使用风电，绿电占比 60\%，年减碳 10 万吨
\end{itemize}

\textbf{当前差距}：国内数据中心绿电占比普遍在 20-40\%，与头部企业的 60-100\% 有差距。建议推动绿电交易和绿证市场，降低绿电获取成本。

\section{算力 - 电力协同调度}
\label{sec:co-scheduling}

\subsection{算力 - 电力耦合建模}

\textbf{概念定义}：算力 - 电力耦合建模是建立算力任务 - 电力消耗 - 碳排放的定量关系模型，是协同调度的基础\cite{dong2022east}。

\textbf{建模粒度}：
\begin{itemize}
    \item \textbf{任务级}：单个算力任务的能耗模型，输入为任务类型、数据量、算法复杂度
    \item \textbf{服务器级}：单台服务器的能耗模型，输入为 CPU/GPU 利用率、内存占用、网络流量
    \item \textbf{集群级}：整个数据中心集群的能耗模型，输入为总负载、冷却系统状态、PUE
    \item \textbf{枢纽级}：东数西算枢纽级的宏观模型，输入为算力规模、电力结构、网络拓扑
\end{itemize}

\textbf{效率指标}：
\begin{itemize}
    \item \textbf{PUE（Power Usage Effectiveness）}：数据中心总能耗/IT 设备能耗，目标值<1.2
    \item \textbf{CUE（Carbon Usage Effectiveness）}：单位 IT 能耗碳排放（kgCO₂/kWh），目标值<0.3
    \item \textbf{WUE（Water Usage Effectiveness）}：单位 IT 能耗水耗（L/kWh），目标值<0.5
    \item \textbf{EUE（Energy Usage Effectiveness）}：综合能效指标，考虑多资源消耗
\end{itemize}

\textbf{主流方法}：

\begin{itemize}
    \item \textbf{物理建模}：基于能量守恒、热力学定律建立物理模型，精度高但复杂。
    
    \textit{优点}：可解释性强、外推能力强、无需数据。
    
    \textit{缺点}：建模复杂、参数多、适应性差。
    
    \item \textbf{数据驱动}：使用机器学习从历史数据学习能耗模型，适应性强但需要数据。
    
    \textit{优点}：适应性强、自动学习、精度较高。
    
    \textit{缺点}：需要大量数据、可解释性差、外推能力弱。
    
    \item \textbf{混合建模}：结合物理模型和数据驱动，平衡精度和适应性。
    
    \textit{优点}：兼顾精度和适应性、可解释性中等。
    
    \textit{缺点}：实现复杂、需要数据和物理知识。
\end{itemize}

\subsection{源荷互动机制设计}

\textbf{概念定义}：源荷互动是指算力负载（荷）响应电力系统信号（源），实现供需协同。这是算力 - 电力协同调度的核心机制\cite{iea2023data}。

\textbf{互动信号}：
\begin{itemize}
    \item \textbf{价格信号}：分时电价、实时电价、尖峰电价
    \item \textbf{碳信号}：碳价、碳配额、绿证价格
    \item \textbf{调度信号}：需求响应指令、调频信号、备用请求
\end{itemize}

\textbf{响应机制}：
\begin{itemize}
    \item \textbf{价格响应}：负载根据电价调整执行时间，高峰时段减少负载
    \item \textbf{碳响应}：负载根据碳强度调整执行时间，高碳时段减少负载
    \item \textbf{调度响应}：负载根据电网调度指令快速调节，参与调频备用
\end{itemize}

\textbf{激励机制}：
\begin{itemize}
    \item \textbf{经济补偿}：参与需求响应的负载获得电费减免或直接补偿
    \item \textbf{市场收益}：参与电力市场交易获得调频、备用等服务收益
    \item \textbf{政策激励}：绿电消纳比例纳入考核，获得政策支持
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{美国 PJM}：数据中心参与调频市场，响应时间<5 分钟，年收益可达投资成本的 10-20\%
    \item \textbf{欧洲 Nord Pool}：数据中心参与日前市场和实时市场，根据电价调整负载
    \item \textbf{中国广东}：数据中心参与需求响应，补偿标准 3-5 元/kWh
\end{itemize}

\textbf{当前差距}：国内电力市场机制尚不完善，数据中心参与渠道有限。建议推动电力市场改革，明确数据中心参与规则和补偿标准。

\subsection{联合优化调度平台}

\textbf{概念定义}：联合优化调度平台是算力 - 电力协同调度的技术载体，需要整合监测、预测、优化、控制、交易等功能\cite{dong2022east}。

\textbf{平台架构}：
\begin{itemize}
    \item \textbf{感知层}：采集算力负载、电力消耗、环境参数等数据
    \item \textbf{预测层}：预测算力需求、电力负荷、可再生能源出力
    \item \textbf{优化层}：求解算力 - 电力联合优化问题，生成调度策略
    \item \textbf{控制层}：执行调度策略，控制算力任务和电力设备
    \item \textbf{交易层}：参与电力市场交易，优化购电成本
\end{itemize}

\textbf{关键技术}：
\begin{itemize}
    \item \textbf{微服务架构}：模块化设计，支持灵活扩展
    \item \textbf{数字孪生}：建立数据中心虚拟模型，支持仿真和优化
    \item \textbf{边缘计算}：在边缘节点实现本地优化，降低时延
    \item \textbf{区块链}：支持绿电溯源和碳足迹追踪
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{Google Carbon-Intelligent Computing}：根据碳强度调度计算任务，高碳时段减少负载，年减碳 100 万吨
    \item \textbf{阿里云能耗管理平台}：整合算力 - 电力数据，实现联合优化，PUE 降至 1.15
    \item \textbf{华为智能数据中心}：使用 AI 优化冷却系统，能耗降低 15\%
\end{itemize}

\textbf{当前差距}：国内联合优化调度平台处于起步阶段，功能完善度和智能化水平与 Google 等有差距。建议加大研发投入，推动平台标准化。

\section{典型应用场景}
\label{sec:scenarios}

\subsection{大模型训练场景}

\textbf{场景特点}：
\begin{itemize}
    \item \textbf{长时任务}：单次训练持续数天至数周
    \item \textbf{高能耗}：单次训练耗电 10-100 万度
    \item \textbf{可中断}：支持断点续训，可迁移
    \item \textbf{资源密集}：需要大量 GPU/TPU 资源
\end{itemize}

\textbf{调度策略}：
\begin{itemize}
    \item \textbf{断点续训}：在电价高或碳强度高的时段暂停训练，低谷时段继续
    \item \textbf{弹性调度}：根据资源可用性动态调整训练 batch size 和学习率
    \item \textbf{跨枢纽迁移}：将训练任务迁移至可再生能源丰富的西部枢纽
\end{itemize}

\textbf{优化效果}：
\begin{itemize}
    \item 训练成本降低 30-50\%
    \item 绿电占比提升 50-80\%
    \item 碳排放减少 40-60\%
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{Meta}：将大模型训练任务调度至水电丰富的挪威数据中心，绿电占比 95\%
    \item \textbf{百度}：使用断点续训 + 电价响应策略，训练成本降低 35\%
\end{itemize}

\subsection{推理服务场景}

\textbf{场景特点}：
\begin{itemize}
    \item \textbf{实时性要求高}：P99 延迟<100ms
    \item \textbf{负载波动大}：日波动可达 5-10 倍
    \item \textbf{SLA 约束}：服务可用性>99.9\%
    \item \textbf{资源分散}：边缘 - 云端协同部署
\end{itemize}

\textbf{调度策略}：
\begin{itemize}
    \item \textbf{负载预测}：提前预测负载峰值，预扩容资源
    \item \textbf{弹性扩缩容}：根据负载自动扩缩容，避免资源浪费
    \item \textbf{边缘协同}：将部分推理任务下沉至边缘，降低云端负载
\end{itemize}

\textbf{优化效果}：
\begin{itemize}
    \item 资源利用率提升至 70-80\%
    \item P99 延迟<100ms
    \item 能耗降低 20-30\%
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{字节跳动}：使用负载预测 + 弹性扩缩容，推理服务资源利用率提升至 75\%
    \item \textbf{腾讯}：边缘 - 云端协同推理，延迟降低 40\%，能耗降低 25\%
\end{itemize}

\subsection{东数西算场景}

\textbf{场景特点}：
\begin{itemize}
    \item \textbf{跨域调度}：东西部枢纽间任务迁移
    \item \textbf{网络时延}：跨域网络时延 20-50ms
    \item \textbf{数据合规}：数据跨境/跨区域流动限制
    \item \textbf{能源优势}：西部可再生能源丰富、电价低
\end{itemize}

\textbf{调度策略}：
\begin{itemize}
    \item \textbf{数据分级}：热数据（实时访问）存放东部，冷数据（归档存储）迁移西部
    \item \textbf{任务 - 数据协同}：计算任务向数据所在区域调度，减少数据传输
    \item \textbf{网络 - 电力联合优化}：综合考虑网络成本和电力成本
\end{itemize}

\textbf{优化效果}：
\begin{itemize}
    \item 跨域带宽成本降低 40-50\%
    \item 综合成本降低 25-35\%
    \item 碳排放减少 30-40\%
\end{itemize}

\textbf{典型案例}：
\begin{itemize}
    \item \textbf{阿里云}：东西部数据中心协同，热/温/冷数据分级存储，带宽成本降低 45\%
    \item \textbf{华为云}：网络 - 算力联合优化，综合成本降低 30\%
    \item \textbf{三大运营商}：国家枢纽节点建设，西部节点绿电占比目标 60\%+
\end{itemize}

\section{挑战与未来方向}
\label{sec:challenge}

\subsection{技术挑战}

\textbf{异构算力标准化}：缺乏统一的算力度量标准，不同厂商指标不可比。需要推动建立类似 SPEC 的算力度量标准。

\textbf{多时间尺度协同}：秒级（推理）、分钟级（训练）、小时级（迁移）的多尺度调度协同困难。需要建立分层调度架构。

\textbf{不确定性处理}：新能源波动 + 算力需求不确定性的联合鲁棒调度。需要发展随机优化和鲁棒优化方法。

\textbf{跨域协同机制}：东西部枢纽间的利益分配、数据合规、网络约束。需要建立跨域协同标准和机制。

\subsection{工程挑战}

\textbf{数据采集与质量}：算力 - 电力数据采集不完整、质量参差不齐。需要建立标准化数据采集框架。

\textbf{实时性要求}：调度决策需要在秒级 - 分钟级完成，对算法效率要求高。需要发展高效优化算法。

\subsection{政策与市场挑战}

\textbf{电力市场机制}：国内电力市场尚不完善，数据中心参与渠道有限。需要推动电力市场改革。

\textbf{绿电交易机制}：绿电交易和绿证市场不成熟，获取成本高。需要完善绿电交易机制。

\textbf{数据合规}：数据跨境/跨区域流动限制影响跨域调度。需要明确数据合规边界。

\section{结论}
\label{sec:conclusion}

本文系统综述了算力调度、电力调度及算力 - 电力协同调度的研究现状、关键技术和发展趋势。通过对 156 篇核心文献的分析，主要发现包括：

\begin{enumerate}
    \item \textbf{算力调度}：异构算力统一度量尚未形成标准，多目标调度算法以强化学习和进化算法为主，跨域协同调度处于起步阶段。
    
    \item \textbf{电力调度}：数据中心电力负荷预测精度已达 95\%+，源网荷储协同调度技术逐步成熟，绿电消纳优化空间巨大。
    
    \item \textbf{协同调度}：算力 - 电力耦合建模处于起步阶段，联合优化调度缺乏统一框架，源荷互动机制待完善。
    
    \item \textbf{场景验证}：大模型训练、推理服务、东数西算三大场景验证初具规模，优化效果显著（成本降低 30\%+、绿电占比 60\%+）。
    
    \item \textbf{指标差距}：当前技术水平与目标值存在显著差距，需要持续技术突破和政策支持。
\end{enumerate}

\textbf{对企业的建议}：
\begin{itemize}
    \item \textbf{技术选型}：优先发展负荷预测、多目标调度等成熟技术
    \item \textbf{平台建设}：逐步建设算力 - 电力联合调度平台
    \item \textbf{绿电获取}：通过自建、采购、交易等方式提升绿电占比
    \item \textbf{市场参与}：积极参与电力市场，获取额外收益
\end{itemize}

\textbf{对研究者的建议}：
\begin{itemize}
    \item \textbf{问题导向}：聚焦企业级应用的实际问题
    \item \textbf{数据共享}：推动数据集和基准测试开源
    \item \textbf{跨学科合作}：与电力系统、优化理论等学科合作
    \item \textbf{标准参与}：参与行业标准制定
\end{itemize}

算力 - 电力协同调度是实现双碳目标和"东数西算"战略的关键技术，需要学术界、产业界、政府共同努力，推动技术创新和产业发展。
