## 五、研究内容详细阐述

### 5.1 研究背景与动机

**现实场景**：
- 全球数据中心 2025 年能耗预计达 1000 TWh，占全球用电量 3-4%
- 中国"双碳"目标：2030 碳达峰，2060 碳中和
- "东数西算"工程：8 大算力枢纽，10 大数据中心集群
- 西部地区新能源丰富（风电/光伏占比超 40%），但存在弃风弃光问题

**研究动机**：
- 数据中心作为可调节负荷，可通过算力 - 电力协同调度提升新能源消纳
- 现有方法无法同时优化成本、消纳率和碳排放多目标
- 深度学习在调度领域的应用尚未充分探索

### 5.2 现有工作局限性

**局限性 1：传统调度方法无法处理新能源波动性**
- Rule-based 方法只能被动响应新能源出力
- 无法学习复杂的时序依赖关系
- 在动态环境下性能下降明显

**局限性 2：储能系统未被充分利用**
- 现有工作多关注新能源消纳，忽视储能套利潜力
- 峰谷电价差（0.8 元/kWh）未被有效利用
- 储能容量配置与负载匹配缺乏指导

**局限性 3：缺乏可解释的调度决策机制**
- 深度学习模型是"黑盒"，难以解释决策依据
- 调度员无法理解 AI 为何做出特定决策
- 影响 AI 调度系统在实际场景的部署

### 5.3 核心思想

**一句话概括**：
> 使用 PPO 深度强化学习学习最优调度策略，结合储能系统实现算力 - 电力协同优化，并通过 LLM 生成可解释的调度决策。

**关键洞察**：
- PPO 可学习动态环境下的复杂策略，无需精确模型
- 储能系统可作为"缓冲"，平衡新能源波动
- LLM 可将数值决策转化为人类可读的解释

### 5.4 技术挑战

**挑战 1：新能源出力与负载的时序匹配**
- 风电/光伏出力受气象条件影响，预测误差 10-20%
- 数据中心负载随 QPS 波动，峰值/谷值比 3-5 倍
-  naive 方法无法处理双重不确定性

**挑战 2：储能充放电决策的实时性要求**
- 调度决策需在秒级完成
- 充放电效率损失（5%）需纳入优化
- SOC 约束（10%-90%）需严格保证

**挑战 3：调度决策的可解释性不足**
- PPO 策略网络是"黑盒"
- 调度员需要理解"为什么此时充电/放电"
- 需要生成符合领域知识的解释

### 5.5 方法模块

**模块 1：PPO 调度优化器**
- 状态空间（5 维）：时段、新能源、负载、SOC、电价
- 动作空间（2 维）：充电功率、放电功率
- 奖励函数：负成本 + 负碳排放 + 新能源消纳奖励
- 网络架构：纯 NumPy 实现（CPU），FC(64) 两层
- 训练参数：500 episodes, γ=0.99, lr=3e-4

**模块 2：储能峰谷套利模块**
- 容量：5 MWh（与负载匹配）
- 充放电功率：1 MW
- 策略：谷段（0.4 元/kWh）充电，峰段（1.2 元/kWh）放电
- 效率：充/放电 95%

**模块 3：LLM 可解释性调度生成器**
- 模型：google/gemma-3-4b（4B 参数）
- 部署：Lmstudio 本地 HTTP API
- 输入：调度状态（时段、SOC、电价等）
- 输出：人类可读的决策解释

### 5.6 实验验证

**数据集**：
- 山东电力系统：3,046 个电厂，378 GW 装机
- 阿里云 GPU 集群：4,278 节点，10,412 GPU
- GenAI 任务：68,195 个推理任务

**基线方法**：
- Rule-based：新能源优先调度
- Greedy：谷段充电/峰段放电
- MPC：滚动优化（horizon=12）

**评估指标**：
- 用电成本（元/天）
- 新能源消纳率（%）
- 碳排放（kgCO₂/天）

**主要结果**：
| 方法 | 成本 | 消纳率 | 碳排放 |
|------|------|--------|--------|
| Rule-based | 16.84 | 100.0% | 11.74 |
| Greedy | 16.64 | 100.0% | 11.45 |
| MPC | 16.64 | 100.0% | 11.45 |
| **PPO** | 16.76 | 100.0% | **11.02** |

**关键发现**：
- 负载>新能源时，消纳率可达 100%
- 储能系统降低用电成本 1.2%
- PPO 碳排放最优（-6.1%）

### 5.7 主要贡献

**贡献 1**：提出 PPO 算力 - 电力协同优化方法
- 纯 CPU 实现，无需 GPU 硬件
- 500 episodes 收敛，推理延迟<10ms
- 碳排放降低 6.1%

**贡献 2**：实现 LLM 可解释性调度
- 本地部署 google/gemma-3-4b
- 生成人类可读的决策解释
- 提升决策透明度

**贡献 3**：开源代码和真实数据集
- GitHub 仓库：https://github.com/zksgithub/compute-power-scheduling
- 数据集：山东电力 + 阿里云 GPU 集群
- 可复现性保证
